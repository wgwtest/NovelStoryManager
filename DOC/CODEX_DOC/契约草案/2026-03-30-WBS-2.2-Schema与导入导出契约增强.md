# WBS 2.2 Schema 与导入导出契约增强

工作目标：

- 在项目底座完成后，补齐公开 schema、导入导出接口和版本迁移规则，让外部结构化数据能稳定进出系统。

工作内容：

- 补齐公开导入 schema 和对应文档。
- 设计并实现完整项目导出与对象级导出能力。
- 明确 schema 版本号和版本迁移策略。

潜在难点：

- 导入导出一旦对外暴露，就要求字段命名和版本策略更稳定。
- 版本迁移如果没有测试覆盖，后续 schema 演进容易破坏旧项目。

Depends On：

- `WBS 2.1` 项目底座实施

成果物：

- 导入 schema 文档
- 导出接口或命令
- 版本迁移说明
- 对应测试用例

验收方法：

- 运行 `npm run test`，预期新增的导入导出相关测试通过。
- 如存在独立 schema 包测试，运行 `npm run test --workspace @novelstory/schema`，预期退出码为 `0`。
- 人工核验仓库中存在导入 schema 文档和版本迁移说明。
