# WBS 2.1 项目底座实施

工作目标：

- 交付首个可运行的项目底座，包括文件化单项目存储、schema 包、本地服务、常驻对象库和知识库主视图。

工作内容：

- 搭建 `npm workspaces` 工程骨架。
- 实现 `packages/schema`、`apps/service`、`apps/web` 的基础运行链路。
- 提供 `fixtures/projects/sample-novel` 示例项目，并在知识库视图中完成打开、查看和编辑。
- 保留 Graph / Tracks 入口，但不在本节点进入复杂交互实现。

潜在难点：

- 文件读写、schema 校验和前端编辑链路需要同时打通，任何一处边界不清都会阻塞整体。
- 需要保证对象事实与视图状态分文件存储，避免后续视图功能污染模型对象。
- 必须保持 AI 理解外置，不能在工具本体里偷偷引入正文解析逻辑。

Depends On：

- `WBS 1.5` 技术方案选型
- `WBS 1.6` 初始任务拆解与时间规划
- `DOC/CODEX_DOC/开发计划/03-WBS-L2-WBS-2.1-项目底座实施计划.md`

成果物：

- `packages/schema/`
- `apps/service/`
- `apps/web/`
- `fixtures/projects/sample-novel/`
- 可运行的对象库侧栏与知识库主视图

验收方法：

- 运行 `npm install`，预期依赖安装成功。
- 运行 `npm run typecheck`，预期退出码为 `0`。
- 运行 `npm run test`，预期退出码为 `0`。
- 启动 `npm run dev:service` 与 `npm run dev:web`，预期本地服务和前端页面均能启动。
- 人工打开示例项目，核验左侧对象库常驻、Knowledge tab 可查看并编辑集合数据、Graph / Tracks tab 仅显示占位入口。
