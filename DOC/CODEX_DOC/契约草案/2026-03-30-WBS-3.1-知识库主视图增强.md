# WBS 3.1 知识库主视图增强

工作目标：

- 在项目底座已有的知识库界面上，补齐当前类型筛选、字段分组和对象引用跳转能力，并让对象库、知识表、检查器围绕同一份对象状态联动。

Owner：

- Codex

Write Scope：

- `apps/web/src/App.tsx`
- `apps/web/src/components/ObjectLibrary.tsx`
- `apps/web/src/components/KnowledgeView.tsx`
- `apps/web/src/components/ObjectInspector.tsx`
- `apps/web/src/App.test.tsx`
- `apps/web/src/styles.css`

工作内容：

- 为对象列表增加类型切换后的筛选能力。
- 为详情页增加 `Core Fields / Additional Fields / References` 分组布局。
- 为对象之间的关联字段提供基础跳转能力。
- 补齐对应前端测试和状态回退逻辑。

潜在难点：

- 过滤结果、当前选中对象和检查器草稿之间容易出现状态不同步。
- 引用跳转需要先解析对象 ID，再同步切换对象类型和当前选中对象。

- `WBS 2.1` 项目底座实施
