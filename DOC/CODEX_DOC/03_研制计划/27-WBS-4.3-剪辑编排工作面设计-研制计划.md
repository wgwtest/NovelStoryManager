## 工作目标

- 交付剪辑编排工作面的父级设计说明与实施分支，明确它为什么采用“剪辑编排”心智，以及章节为什么只是观察结果而不是建模前提。
- 重点先证明轨道、块和拖拽编排能力真实存在。

## 工作内容

- 先解释剪辑编排对共享对象真源、轨道预设和章节切片状态的读取边界。
- 先解释时间线、空间线和剧情线为何能进入同一套轨道体系。
- 先定义轨道、块、时间尺、空间线和预设区。
- 再验证入轨、换轨、排序、拖拽、缩放和平移等基础能力。
- 再验证预设保存、加载和对象真源不被污染。

## 实施方式

- 按“要素设计 -> 能力验证 -> 功能验证 -> 样式验证”的顺序推进。
- 剪辑编排视图能力表达必须贴近剪辑软件式剪辑编排心智，而不是只做静态列表。
