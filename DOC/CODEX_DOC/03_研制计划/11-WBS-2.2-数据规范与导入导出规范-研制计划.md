# WBS 2.2 数据规范与导入导出规范

## 基本信息

- `WBS` 编号：`WBS 2.2`
- 上级节点：`WBS 2`
- 当前状态：`开发中`

## 前置条件

- `WBS 2.1` 已完成项目底座和文件化存储基础链路。

## 后续节点

- `WBS 3` 三视图共性机制
- `WBS 4` 知识库工作台
- `WBS 5` 关系图工作台
- `WBS 6` 多轨编排视图

## 工作目标

- 固化项目内部文件规范、项目级导入导出规范和对象级导入导出规范，补齐一个小型但字段覆盖尽量全面的样例数据，并用硬测试保证规则可被验证。

## 工作内容

- 编写项目数据规范与导入导出规范正文。
- 在 `schema` 层实现项目级和对象级 bundle 的结构校验。
- 在 `service` 层实现导入、导出和往返一致性校验。
- 提供正样例、反样例、版本不兼容样例和一份高字段覆盖样例数据。
- 重写验收口径，避免在用户人工验收清单中出现读代码和 `npm` 命令项。

## 实施方式

- 按“规范正文 -> schema 实现 -> service 实现 -> 样例夹具与回归测试”的顺序推进。
- 验证必须覆盖正样例、反样例、坏引用、重复对象和版本不兼容场景。
- 用户人工验收只保留可观察文件、样例数据和自测证据核验；命令执行进入自测报告，不再写入人工验收清单。
- 历史细化计划已归档到 [04-WBS-L2-WBS-2.2-数据规范与导入导出规范计划.md](/home/wgw/CodexProject/NovelStoryManager/DOC/CODEX_DOC/06_过程文档/02_历史计划/04-WBS-L2-WBS-2.2-数据规范与导入导出规范计划.md)。

## 验收方法

- 人工核验规范正文已存在，并至少包含“内部项目文件规范”“项目级导入导出 bundle”“对象级导入导出 bundle”“版本与失败规则”四节。
- 人工核验 `fixtures/import-export/` 下存在本节点约定的正样例、反样例和 `sample-rich-project.json`，且该样例能覆盖角色、地点、势力、事件、关系引用和视图状态等主要字段。
- 查看对应自测报告，确认其中已执行 `npm run test --workspace @novelstory/schema`、`npm run test --workspace @novelstory/service`、`npm run typecheck`、`npm run build`、`npm run test`，且结果均为成功。
- 自测必须覆盖项目级 bundle、对象级 bundle、缺字段、坏引用、版本不兼容、重复对象和往返一致性场景。
- 后续生成的用户人工验收清单中，不再出现“手工读代码确认接口存在”或“用户自行执行 npm 命令”这类条目。
