## 工作目标

- 用完整操作链验证 `BaseLab` 中的卷宗工作面已经能承担对象浏览与维护，并用可审阅的说明层验证“为什么这样做”，而不是只通过单点能力测试。

## 工作内容

- 设计并验证典型操作链路，例如“筛选对象 -> 查看字段 -> 点击引用 -> 切换目标对象 -> 返回继续编辑”。
- 补齐“关键字筛选 -> 自动切换首个匹配对象 -> 引用回查返回”和“审校队列 -> 缺失字段提示 -> 检查器字段观察”两条完整用户链路。
- 补齐“设计说明 -> 样例规范 -> 工作面验证”的审阅链路。
- 补齐“人物卷宗 -> 关系档案 -> 打开关系卷宗 -> 编辑 source/target/summary”和“人物卷宗 -> 出场记录 -> 章节切片回查”两条事实层链路。
- 基于模拟数据验证卷宗目录、正文、检查器三者之间的数据一致性。
- 补齐用户人工验收清单和自测报告。
- 回归独立卷宗页中的对象库侧栏与卷宗主视图的数据一致性。

## 实施方式

- 功能验证文档和 `BaseLab` 代码回归一起推进。
- 验收清单只写用户可观察链路，自测报告承接命令与测试结果。
- 本节点完成后，卷宗工作面的主工作台接入仍由 `WBS 4.1.5` 承接。
