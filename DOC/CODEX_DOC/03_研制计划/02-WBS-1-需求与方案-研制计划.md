# WBS 1 需求与方案

## 基本信息

- `WBS` 编号：`WBS 1`
- 上级节点：`WBS 0`
- 当前状态：`开发中`

## 前置条件

- `WBS 0` 已建立顶层研发路线和文档归档规则。

## 后续节点

- `WBS 1.1` 产品边界确认与文档归档
- `WBS 1.2` 信息架构与页面分解
- `WBS 1.3` 低保真原型设计
- `WBS 1.4` 高风险交互原型验证
- `WBS 1.5` 技术方案选型
- `WBS 1.6` 初始任务拆解与时间规划
- `WBS 1.7` 中高保真原型设计与生成方案
- `WBS 1.8` BaseLab验证工具构建方案
- `WBS 2.1` 项目底座实施

## 工作目标

- 在进入实现前，把产品边界、页面结构、原型目标、交互风险、技术栈和初始时间规划全部收敛清楚，并在新增评阅意见要求下补入中高保真原型基线与跨节点验证工具方案。

## 工作内容

- 固化单项目、文件化、对象库真源和 AI 外置等产品边界。
- 明确左侧对象库常驻、三主视图切换和右侧检查器的工作台结构。
- 通过原型和交互验证明确高风险拖拽能力的真实目标。
- 补入中高保真原型生成方案，为后续界面细化提供更清晰的视觉基线。
- 补入 `BaseLab` 验证工具构建方案，为后续跨节点技术验证提供独立入口与边界规则。
- 基于原型目标冻结技术方案，再拆出首轮可执行任务与时间安排。

## 实施方式

- 按 `WBS 1.1 -> 1.2 -> 1.3 -> 1.4 -> 1.5 -> 1.6` 完成首轮规划。
- `WBS 1.7` 作为评阅后新增的补充节点，承接中高保真原型需求，不回改已验收节点编号。
- `WBS 1.8` 作为评阅后新增的补充节点，承接跨节点验证工具方案需求，不回改已验收节点编号。
- 设计正文进入 `02_设计说明/`，本目录只保留节点级执行说明。
- 完成 `WBS 1.6` 后，才能正式进入 `WBS 2` 的实现节点。
- `WBS 1.7` 不回滚已开始的 `WBS 2-7`，但会约束后续界面细化、节点风格和观察输出展示目标。
- `WBS 1.8` 同样属于低耦合 Prep，不直接回滚现有实现节点，但会约束后续验证工具的组织方式。
