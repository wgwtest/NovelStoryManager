# WBS 3 多视图共性机制

## 基本信息

- `WBS` 编号：`WBS 3`
- 上级节点：`WBS 0`
- 当前状态：`开发中`

## 前置条件

- `WBS 2.1` 已完成项目底座实施。
- `WBS 2.2` 已明确数据规范、导入导出规范和样例数据要求。

## 后续节点

- `WBS 3.1` 交互画布与渲染基座选型
- `WBS 3.2` 通用视图元素与坐标系统
- `WBS 3.3` 通用编辑操作与存储协议
- `WBS 3.4` 通用验证基线
- `WBS 4` 视图设计

## 工作目标

- 在正式视图分支展开前，先冻结它们共享的交互底座、元素表示、坐标体系、存储协议和验证方法，避免各视图各做一套后再返工。

## 工作内容

- 明确多种建模视图中哪些能力共享，哪些能力允许分化。
- 明确节点、关系边、轨道块、选区、视口、坐标等基础表示。
- 明确拖拽、缩放、平移、选择、属性编辑、保存等通用操作边界。
- 明确多视图共同验证基线，以及用户人工验收与开发者自测的分工。

## 实施方式

- 先做 `3.1` 技术底座决策，再做 `3.2` 与 `3.3` 的模型和协议定义，最后以 `3.4` 固化验证基线。
- `WBS 4` 的各执行分支必须依赖本节点产出，不再跳过共性层直接做工作台 demo。
