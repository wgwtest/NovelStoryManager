# WBS 4 视图设计

## 基本信息

- `WBS` 编号：`WBS 4`
- 上级节点：`WBS 0`
- 当前状态：`开发中`

## 前置条件

- `WBS 3` 已冻结多视图共性机制。
- `WBS 1.7.2` 已提供四张核心工作面原型，用于约束后续视图设计方向。

## 后续节点

- `WBS 4.1` 卷宗工作面设计
- `WBS 4.2` 蓝图推演工作面设计
- `WBS 4.3` 剪辑编排工作面设计
- `WBS 4.4` 场景调度工作面设计
- `WBS 7` 观察输出层

## 工作目标

- 在共享对象真源和共享控制契约之上，建立 `卷宗 / 蓝图推演 / 剪辑编排 / 场景调度` 四个严格解耦的工作面设计父节点。
- 明确回答“同一套数据如何驱动多个可独立替换的视图”，为后续实现提供总设计约束。

## 工作内容

- 统一维护四个工作面分支的父子关系、依赖关系和命名规则。
- 补齐 `WBS 4` 总设计说明，明确模型真源、视图状态、`MVC` 分层和视图解耦原则。
- 保证四个分支都复用 `WBS 3` 的共性层，不各自重建底座。
- 保证四个分支都能回接 `WBS 1.7.2` 当前冻结的四张核心工作面原型。
- 保证四个工作面只通过共享模型与控制器协作，不在视图实现层直接互相依赖。

## 实施方式

- 先收口父节点，再进入 `4.1 / 4.2 / 4.3 / 4.4` 四个执行分支。
- 每个执行分支默认继续拆为“要素设计 -> 能力验证 -> 功能验证 -> 样式验证”。
