## 工作目标

- 在共享对象真源和共享控制契约之上，建立“多视图设计”父节点，当前首批工作面为 `卷宗 / 蓝图推演 / 剪辑编排 / 场景调度`。
- 明确回答“同一套数据如何驱动多个可独立替换的视图”，并把“视图推进中允许增量补模”写成后续实现的总设计约束。

## 工作内容

- 统一维护多视图分支的父子关系、依赖关系和命名规则，明确当前首批四个工作面不是最终上限。
- 补齐 `WBS 4` 总设计说明，明确模型真源、视图状态、`MVC` 分层、扩展性约束和模型增量补齐策略。
- 保证当前首批四个分支都复用 `WBS 3` 的共性层，不各自重建底座。
- 保证当前首批四个分支都能回接 `WBS 1.7.2` 当前冻结的四张核心工作面原型。
- 保证各工作面只通过共享模型与控制器协作，不在视图实现层直接互相依赖。
- 当前阶段优先收口 `WBS 4.1 卷宗`，其验证链路先走 `BaseLab + 模拟数据`，再回接主工作台。

## 实施方式

- 先收口多视图父节点，再进入当前首批 `4.1 / 4.2 / 4.3 / 4.4` 四个执行分支。
- 每个执行分支默认继续拆为“要素设计 -> 能力验证 -> 功能验证 -> 样式验证”。
- `WBS 4.1` 作为当前主线，先完成独立验证，再进入主工作台融合。
