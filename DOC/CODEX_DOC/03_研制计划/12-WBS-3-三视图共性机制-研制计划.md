## 工作目标

- 在三主视图正式展开前，先冻结它们共享的交互底座、元素表示、坐标体系、存储协议和验证方法，避免各视图各做一套后再返工。

## 工作内容

- 明确 `Knowledge / Graph / Tracks` 哪些能力共享，哪些能力允许分化。
- 明确节点、关系边、轨道块、选区、视口、坐标等基础表示。
- 明确拖拽、缩放、平移、选择、属性编辑、保存等通用操作边界。
- 明确三视图的共同验证基线，以及用户人工验收与开发者自测的分工。

## 实施方式

- 先做 `3.1` 技术底座决策，再做 `3.2` 与 `3.3` 的模型和协议定义，最后以 `3.4` 固化验证基线。
- `WBS 4-6` 的 `Core` 实现必须依赖本节点产出，不再跳过共性层直接做工作台 demo。
