# WBS 1.1 产品边界确认与文档归档

## 节点定位

1. 上级节点：`WBS 1. 需求与方案`
2. 节点类型：`Prep`
3. 当前状态：`已完成`

## 节点目标

完成 NovelStoryManager 的产品边界确认、文档根归档、设计稿归档和 WBS 主计划固化，使后续选型与实施都挂在稳定结构上，而不是继续依赖临时聊天上下文。

## 本节点输出物

1. 项目文档根 `DOC/CODEX_DOC/`
2. 文档索引 README
3. 本地工程策略映射文档
4. 工程总体分析文档
5. WBS L0 研发总纲
6. 迁移后的设计稿

## 本节点完成标准

1. 文档根建立完成
2. 设计稿归档到稳定目录
3. 当前产品边界可直接从文档读取
4. WBS 顶层结构明确
5. 后续选型与实施计划的归档位置明确

## 下一节点建议

1. `WBS 1.2` 技术方案选型
2. `WBS 1.3` 初始任务拆解与时间规划
3. `WBS 2.1` 项目底座实施计划

## 当前约束

1. 不替代技术方案选型
2. 不跨越 `WBS 1.2` 和 `WBS 1.3` 直接进入 Core 开发
3. 本节点产物以“边界和归档清晰”为准，不承载技术决策
