# 档案阅览型 方案说明

## 套系定位

把世界模型当作可检索资料来查阅和校读，强调索引、正文页框和脚注引用。

## 结构性标记

- `档案索引架`
- `引文页框`
- `参考脚注列`

## 外部参照

- [Airtable Interface Designer](https://support.airtable.com/v1/docs/getting-started-with-airtable-interface-designer)
- [Obsidian Canvas](https://obsidian.md/canvas)

## 设计转译

1. 产品标题统一使用 `知识编辑器`，项目 `青云劫火录` 只做上下文。
2. 左侧对象库保持长期驻留，但中区主工作面的隐喻完全围绕 `档案阅览型` 重构。
3. 右侧检查器只承载当前对象的辅助详情，不替代中区主工作流。
4. `档案阅览型` 的目标不是换色，而是改变用户处理对象、关系、轨道与切片的方式。
