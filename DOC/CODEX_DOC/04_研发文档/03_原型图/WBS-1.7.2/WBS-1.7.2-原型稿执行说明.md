# WBS 1.7.2 原型稿执行说明

## 目标

在统一语义基线下，为主工作台产出 5 套完整界面方案，供后续比较和人工评阅。

## 统一约束

1. 产品标题统一使用 `知识编辑器`。
2. 项目名 `青云劫火录` 只作为次级上下文显示。
3. 三个主视图统一使用中文：`知识库`、`关系图`、`轨道`。
4. 左列统一为 `对象库`，右列统一为 `检查器`。
5. 主背景统一为白色中性工作台，不使用暖黄底。
6. 左列顺序统一为：搜索框、分类切换、计数与动作、对象列表。
7. 对象库统一覆盖九类对象：人物、势力、地点、物品、境界体系、事件、关系、线索、剧情弧。

## 套系划分

1. `01_卷宗案台型`
   - 强调卷宗索引、字段矩阵和记录详情抽屉。
2. `02_筹划墙型`
   - 强调拖拽筹划墙、灵感便签与素材归堆区。
3. `03_蓝图推演型`
   - 强调蓝图总线、推演节点与端口规则板。
4. `04_剪辑编排型`
   - 强调编排时间尺、轨道头矩阵与待排事件篮。
5. `05_总控指挥型`
   - 强调全局覆盖总览、联动监视窗与输出切片指挥板。

## 每套方案最少文件

1. `01-总工作台总览.pen`
2. `02-知识库页.pen`
3. `03-关系图页.pen`
4. `04-轨道页.pen`
5. `05-对象库与检查器.pen`
6. 对应导出图片文件
7. `方案说明.md`

## 当前执行结果

1. 已完成 `01_卷宗案台型` 到 `05_总控指挥型` 共 5 套目录落盘
2. 每套目录当前均包含 `5` 个 `.pen` 原稿、`5` 个对应 `.png` 导出图和 `1` 份 `方案说明.md`
3. 当前导出图统一由 `Pencil` 以 `3200 x 2000` PNG 输出

## 评阅重点

1. 是否清楚体现左中右三列职责。
2. 是否维持中文化、紧凑化和编辑器气质。
3. 是否让 `知识库`、`关系图`、`轨道` 的含义一眼可分。
4. 五套方案是否真正在主工作面结构上拉开，而不是只换配色。
5. 是否能为后续真实实现提供稳定参考，而不是只做展示海报。
