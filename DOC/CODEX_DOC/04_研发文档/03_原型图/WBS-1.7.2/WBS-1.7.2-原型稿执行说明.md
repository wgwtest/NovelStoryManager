# WBS 1.7.2 原型稿执行说明

## 目标

在统一语义基线下，为主工作台产出 `8` 套完整界面方案，供后续比较和人工评阅。

## 统一约束

1. 产品标题统一使用 `知识编辑器`。
2. 项目名 `青云劫火录` 只作为次级上下文显示。
3. 三个主视图统一使用中文：`知识库`、`关系图`、`轨道`。
4. 左列统一为 `对象库`，右列统一为 `检查器`。
5. 主背景统一为白色中性工作台，不使用暖黄底。
6. 左列顺序统一为：搜索框、分类切换、计数与动作、对象列表。
7. 对象库统一覆盖九类对象：人物、势力、地点、物品、境界体系、事件、关系、线索、剧情弧。

## 五张图的职责拆分

1. `01-总工作台总览.pen`
   - 负责说明整套路线的工作台骨架和三主视图入口。
2. `02-知识库页.pen`
   - 负责说明结构化真值编辑在这套路线上如何落地。
3. `03-关系图页.pen`
   - 负责说明对象、关系、事件、线索的图结构表达。
4. `04-轨道页.pen`
   - 负责说明多时间线、多空间线和观察切片的排布方式。
5. `05-对象库与检查器.pen`
   - 负责说明左侧对象导航和右侧详情联动，而不是重复中区母版。

同一套方案中的五张图必须共享设计语言，但不能复用同一版式机械铺满。

## 套系划分

1. `01_卷宗案台型`
   - 强调卷宗索引、字段矩阵和记录详情抽屉。
2. `02_档案阅览型`
   - 强调档案索引架、引文页框与参考脚注列。
3. `03_蓝图推演型`
   - 强调蓝图总线、推演节点与端口规则板。
4. `04_剧情弧矩阵型`
   - 强调剧情弧矩阵、角色交叉轴与事件覆盖格。
5. `05_剪辑编排型`
   - 强调编排时间尺、轨道头矩阵与待排事件篮。
6. `06_场景调度型`
   - 强调场景舞台图、地点调度层与空间切换条。
7. `07_双镜对读型`
   - 强调双镜对读台、左右对照窗与差异摘要带。
8. `08_切片汇编型`
   - 强调切片汇编台、片段编排列与输出装配栏。

## 每套方案最少文件

1. `01-总工作台总览.pen`
2. `02-知识库页.pen`
3. `03-关系图页.pen`
4. `04-轨道页.pen`
5. `05-对象库与检查器.pen`
6. 对应导出图片文件
7. `方案说明.md`

## 当前执行结果

1. 已完成 `01_卷宗案台型` 到 `08_切片汇编型` 共 `8` 套目录落盘。
2. 每套目录当前均包含 `5` 个 `.pen` 原稿、`5` 个对应 `.png` 导出图和 `1` 份 `方案说明.md`。
3. 当前总计已输出 `40` 张 PNG 设计图。
4. `筹划墙型` 与 `总控指挥型` 已不再作为当前有效路线保留。
5. 当前导出图统一由 `Pencil` 以 `3200 x 2000` PNG 输出。

## 评阅重点

1. 是否清楚体现左中右三列职责。
2. 是否维持中文化、紧凑化和编辑器气质。
3. 是否让 `知识库`、`关系图`、`轨道` 的含义一眼可分。
4. 每套方案内的 `5` 张图是否确实对应 `5` 个页面，而不是同一版式的机械复写。
5. 八套方案是否真正在主工作面结构上拉开，而不是只换配色。
6. 是否能为后续真实实现提供稳定参考，而不是只做展示海报。
