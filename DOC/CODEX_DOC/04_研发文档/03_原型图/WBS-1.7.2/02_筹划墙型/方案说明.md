# 筹划墙型 方案说明

## 套系定位

把剧情组织还原成拖拽筹划墙，中区是大画布，左侧对象库更像素材仓。

## 结构性标记

- `筹划墙画布`
- `灵感便签`
- `素材归堆区`

## 外部参照

- [Milanote Visual Boards](https://milanote.com/templates/product-management/kanban-board)
- [Obsidian Canvas](https://obsidian.md/canvas)

## 设计转译

1. 产品标题统一使用 `知识编辑器`，项目 `青云劫火录` 只做上下文。
2. 左侧对象库保持长期驻留，但中区主工作面的隐喻完全围绕 `筹划墙型` 重构。
3. 右侧检查器只承载当前对象的辅助详情，不替代中区主工作流。
4. `筹划墙型` 的目标不是换色，而是改变用户处理对象、关系、轨道与输出切片的方式。
