# 卷宗案台型 方案说明

## 套系定位

把对象建模当作卷宗整理工作，主工作区强调记录索引、字段矩阵和详情抽屉。

## 结构性标记

- `卷宗目录`
- `字段矩阵`
- `记录详情抽屉`

## 外部参照

- [Airtable Interface / Record Detail](https://support.airtable.com/docs/es/airtable-interface-layout-record-detail)
- [Scrivener Corkboard](https://www.literatureandlatte.com/learn-and-support/video-tutorials/organising-5-get-to-know-the-corkboard)

## 设计转译

1. 产品标题统一使用 `知识编辑器`，项目 `青云劫火录` 只做上下文。
2. 左侧对象库保持长期驻留，但中区主工作面的隐喻完全围绕 `卷宗案台型` 重构。
3. 右侧检查器只承载当前对象的辅助详情，不替代中区主工作流。
4. `卷宗案台型` 的目标不是换色，而是改变用户处理对象、关系、轨道与输出切片的方式。
