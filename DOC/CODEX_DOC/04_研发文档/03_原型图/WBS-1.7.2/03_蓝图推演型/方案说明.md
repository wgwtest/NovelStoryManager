# 蓝图推演型 方案说明

## 套系定位

把对象与关系当作可推演的图结构，中区主角是蓝图总线与节点依赖。

## 结构性标记

- `蓝图总线`
- `推演节点`
- `端口规则板`

## 外部参照

- [Unreal Blueprint Editor](https://dev.epicgames.com/documentation/fr-fr/unreal-engine/blueprints-visual-scripting-editor-user-interface-for-level-blueprints-in-unreal-engine)
- [Obsidian Graph](https://obsidian.md/help/plugins/graph)

## 设计转译

1. 产品标题统一使用 `知识编辑器`，项目 `青云劫火录` 只做上下文。
2. 左侧对象库保持长期驻留，但中区主工作面的隐喻完全围绕 `蓝图推演型` 重构。
3. 右侧检查器只承载当前对象的辅助详情，不替代中区主工作流。
4. `蓝图推演型` 的目标不是换色，而是改变用户处理对象、关系、轨道与切片的方式。
