# WBS 1.7 Pencil 设计稿建议分析

## 评论来源

1. GitHub issue：`#38 WBS 1.7 中高保真原型设计与生成方案`
2. 评论编号：`4183730966`
3. 评论时间：`2026-04-03T14:33:31Z`
4. 评论原意：
   - 认为使用 `Pencil` 创建高保真截面图是个好主意
   - 希望在当前 `WBS 1.7` 下创建子节点
   - 子节点用于承接方案草稿、`Pencil` 设计稿等内容
   - 需要先判断该建议是否合理

## 现状核对

1. 当前 `WBS 1.7` 的本地文档与远端 issue 仍是单节点薄结构，同时承载：
   - 自动生成工具选型
   - 三视图中高保真目标定义
   - 输入包与输出要求
2. 当前主推荐方案是：
   - `Uizard Autodesigner`
   - `Visily`
   - `Figma AI Prototype / Make`
3. 当前方案强调“自动生成高保真草稿”，但还没有把“人工精修设计稿”单独拆出来。
4. 这会导致 `WBS 1.7` 同时承担“路线判断”和“设计稿产出”两类不同工作对象，不利于后续树形理解与验收分层。

## Pencil 方案判断

### 合理部分

1. 将 `Pencil` 纳入 `WBS 1.7` 的补充路线是合理的。
2. 原因：
   - `Pencil` 属于本地化、文件化、开源的 GUI 原型/图稿工具，契合本项目偏本地资产管理的协作习惯。
   - 它适合做页面分区草稿、截面图、局部工作台排版和节点布局说明。
   - 当我们已经有了结构化输入和风格约束后，`Pencil` 可以承担“把自动生成草稿收束成可审阅设计稿”的角色。

### 不合理部分

1. 不应把 `Pencil` 直接替换为 `WBS 1.7` 的主方案。
2. 原因：
   - `Pencil` 不承担“自动生成高保真原型”的核心能力，无法直接覆盖当前 `WBS 1.7` 的生成目标。
   - 它更偏手工绘制与拼装，不适合作为“先快速出多页面候选稿”的唯一入口。
   - 当前项目对 `Graph` 的目标包含 Unreal 蓝图式节点、端口、吸附反馈等高交互表达，`Pencil` 更适合作为静态设计稿与截面图工具，而不是主要生成器。
   - 从官方站点看，当前稳定版仍停留在 `3.1.1`，发布时间较早，说明它更适合作为稳定补充工具，而不是需要大幅依赖新能力的主路线。

## WBS 结构判断

1. 该意见合理，`WBS 1.7` 应当从单节点薄结构继续拆成子节点。
2. 但不建议一次拆出过多空节点。
3. 当前采用“最小但清晰”的两级拆分更合适：
   - `WBS 1.7.1 中高保真原型草稿方案细化`
   - `WBS 1.7.2 Pencil截面图与页面设计稿`

## 拆分理由

### WBS 1.7.1 中高保真原型草稿方案细化

承载的真实工作对象：

1. 收敛自动生成与人工精修的边界
2. 整理三主视图的生成输入包与草稿要求
3. 明确哪些图由生成器先出，哪些局部需要人工工具补绘

### WBS 1.7.2 Pencil截面图与页面设计稿

承载的真实工作对象：

1. 基于 `WBS 1.7.1` 的输入包与草稿结果，产出 `Pencil` 设计稿
2. 重点补齐对象库、工作台截面、关系图节点区、轨道区等局部高保真截面图
3. 形成后续人工评阅和视觉校正使用的稳定设计资产

## 本轮结论

1. `Pencil` 建议总体合理，但定位应为补充路线，不应取代当前“自动生成高保真草稿”的主目标。
2. 当前最合理的处理方式不是直接改写 `WBS 1.7` 主推荐，而是把 `WBS 1.7` 提升为父节点，下挂：
   - `WBS 1.7.1` 负责草稿方案细化
   - `WBS 1.7.2` 负责 `Pencil` 设计稿产出
3. 本轮执行应包括：
   - 更新 `WBS 1.7` 父节点文档
   - 新增 `WBS 1.7.1` 与 `WBS 1.7.2` 本地计划文档
   - 同步创建 GitHub 子 issue，并挂到 `#38` 下
   - 在 `#38` 评论中回写本轮判断与拆分结果
