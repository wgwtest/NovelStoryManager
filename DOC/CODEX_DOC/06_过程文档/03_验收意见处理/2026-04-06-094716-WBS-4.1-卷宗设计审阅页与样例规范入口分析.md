# WBS 4.1 卷宗设计审阅页与样例规范入口分析

## 意见来源

1. `#44 / 4189911724`
2. 评论时间：`2026-04-06 01:24:24 UTC`
3. 评论人：`hugowangguowei`

原始意见可归纳为：

1. 当前 `WBS 4.1.2 / 4.1.3 / 4.1.4` 的验收清单虽然基本合理，但内容密度不满足要求。
2. 当前卷宗独立验证页只展示了操作验证，没有向审阅者解释卷宗的设计思路、功能边界与数据前提。
3. 页面中缺少对样例数据格式规范的直接展示入口，导致“页面能加载什么、为什么这样建模、对象类型为什么这样划分”都无法直观审查。
4. 用户希望先看到一版“页面清单 / 功能清单”，确认方向后再继续深化，而不是直接把现有验证页当作阶段性成品。

## 为什么这条意见重要

这条评论不是在否定 `BaseLab` 卷宗页当前已经实现的筛选、回查、审校和检查器联动。

它真正指出的是另一个问题：

1. 当前页面更像“内部验证台”
2. 但用户当前需要的其实是“可审阅的独立验证页”

两者的差异在于：

1. 内部验证台强调功能是否成立
2. 可审阅验证页除了功能成立，还必须让审阅者直接看懂：
   - 卷宗为什么存在
   - 卷宗覆盖哪些对象类型
   - 对象之间有哪些典型链接关系
   - 页面依赖哪些样例数据与视图状态
   - 当前样例数据需要满足什么格式规范

因此，这条评论的优先级高于单纯补几个交互测试。

## 与现状的对照验证

对照当前代码与文档，评论成立：

1. `DOC/CODEX_DOC/02_设计说明/WBS-4.1-卷宗工作面设计.md` 已经解释了卷宗定位、边界和数据来源，但这些说明只存在于文档中，没有投影到 `BaseLab` 页面。
2. `apps/base-lab/src/components/DossierAbilityLab.tsx` 当前主要承载对象库、卷宗正文、检查器和审校队列，没有“设计说明入口”“功能清单入口”“样例数据规范入口”。
3. 当前页面也没有直接展示：
   - 对象类型为什么分为人物、宗门、地点、物品、体系、事件、关系、线索、主线
   - 样例项目中 `objects/*.json` 与 `views/*.json` 的职责分层
   - 常见引用关系如何从样例对象中落到卷宗按钮
4. 之前的本地分析 `2026-04-06-005608-WBS-4-卷宗优先与多视图增量建模意见分析.md` 只处理到了“卷宗优先、BaseLab 先验、4.1.5 融合”这一层，未覆盖这条更晚出现的评论。

## 对之前同步遗漏原因的判断

之前之所以没有同步到这条意见，原因不是理解错误，而是同步窗口不完整：

1. 早前处理 `#44` 时主要收敛了 `4189174646 / 4189175401`
2. `4189911724` 是更晚出现的追加评论
3. 后续又直接进入了卷宗独立验证页实现与验收覆盖补强
4. 结果是“实现链路推进了，但审阅页这一层要求没有被拉回计划主线”

因此这次需要显式补一层，而不是继续假设当前页面已经满足 `#44` 全部要求。

## 合理的处理方向

从技术和产品边界看，这条意见合理，且可在当前 `WBS 4.1.2 / 4.1.3 / 4.1.4` 范围内处理，不必提前跳到 `WBS 4.1.5`。

更合适的修正方式不是推翻现有卷宗页，而是在其上补一层“审阅导向的解释入口”：

1. 保留当前卷宗操作验证页，继续承载筛选、回查、审校和字段改写。
2. 在同一实验入口下增加可切换的审阅页或审阅面板，至少覆盖：
   - 设计思路
   - 功能清单
   - 对象类型覆盖说明
   - 典型链接关系说明
   - 样例数据格式规范入口
3. 样例数据格式规范入口应能直接展示：
   - `objects/*.json` 与 `views/*.json` 的职责差异
   - 至少一个对象样例结构
   - 至少一个视图状态样例结构
4. 页面排版要从“内部调试台”向“可审阅说明页 + 可操作验证页”收敛。

## 推荐落地方案

推荐采用“单入口、双层内容”的方式，而不是新开一个完全独立实验页：

1. 入口仍保留为 `WBS 4.1 卷宗独立验证`
2. 进入后提供至少两个一级切换：
   - `工作面验证`
   - `设计说明`
3. `设计说明` 内再按信息密度拆成若干审阅卡片或分段：
   - 设计目的
   - 功能清单
   - 对象类型与覆盖边界
   - 典型链接关系
   - 样例数据规范
4. `样例数据规范` 应提供可直接阅读的 JSON 片段或结构清单，而不是只给文档路径。

这样做的理由：

1. 不会推翻现有 `BaseLab` 验证成果
2. 能直接响应“需要一个页面介绍设计思路”和“需要一个按钮显示样例数据格式规范”
3. 仍然保持 `WBS 4.1.2 / 4.1.3 / 4.1.4` 属于独立验证范围

## 对计划的影响

若按该意见纠偏，则当前 `WBS 4.1.2 / 4.1.3 / 4.1.4` 的验收口径需要补充：

1. 不再只验操作链
2. 还要验“设计说明入口是否存在、样例规范是否可见、功能清单是否完整”

这意味着下一轮应先补：

1. `BaseLab` 卷宗独立验证页的信息架构
2. 对应自动化测试与人工验收项
3. `19 / 20 / 21` 三份研制计划中的验收方法描述

在这三项完成前，不宜声称 `#44` 已被完整处理。
