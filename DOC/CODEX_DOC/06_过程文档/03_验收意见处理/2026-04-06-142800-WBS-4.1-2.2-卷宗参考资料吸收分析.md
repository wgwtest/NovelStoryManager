# WBS 4.1 / WBS 2.2 卷宗参考资料吸收分析

## 来源

- 参考资料：`DOC/CODEX_DOC/02_设计说明/参考资料/novel-dossier-system-design.md`
- 关联节点：
  - `WBS 4.1 卷宗工作面设计`
  - `WBS 2.2 数据规范与导入导出规范`

## 任务理解

本轮不是照抄参考资料，也不是推翻当前实现重来，而是：

1. 吸收其中对卷宗定位、统一结构、关系、时间线、证据、完整度等方面更成熟的表达。
2. 修正我当前 `WBS 4.1` 设计说明过浅、过窄的问题。
3. 将真正有助于后续模型演进的数据口径沉淀进 `WBS 2.2`。
4. 保持既有前提不被破坏：
   - 对象库是真源
   - 视图状态独立
   - AI 理解外置
   - 多视图共享数据但不共享实现

## 参考资料中应吸收的部分

### 1. 卷宗不是样式，而是信息组织方式

这一点应直接吸收到 `WBS 4.1`，因为它能解决“卷宗为什么存在”的产品定义问题。

### 2. 统一抽象模型

参考资料提出的：

- 元信息
- 摘要
- 属性
- 时间线
- 关系
- 事件
- 证据
- 分析
- 备注

这套模型是有价值的，但应当被解释为：

`卷宗观察模型`

而不是：

`单个卷宗文件必须一次性存满的字段集合`

### 3. 关系记录必须可描述

参考资料把关系从“链接”提升到“带状态、强度、锚点、说明的关系记录”，这与当前 `relations.json` 路线高度一致，应吸收到 `WBS 2.2` 中作为增强口径。

### 4. 时间线、证据、完整度、异常检测

这四块非常重要，因为它们回答了“卷宗为什么不该只是卡片或表单”。

但它们在当前项目里必须区分：

1. 哪些是事实真源
2. 哪些是观察投影
3. 哪些是派生结果

## 参考资料中不应直接照搬的部分

### 1. 全站一级导航

参考资料的“档案馆总览 / 分析中心 / 证据中心 / 设定中心 / 异常检查 / 导入记录”更接近整站 IA，不应直接塞入 `WBS 4.1` 卷宗工作面设计说明。

### 2. 把时间线当成卷宗本体文件

当前项目已经明确：

- 多时间线、多轨道是观察和编排问题
- 事件对象是真实事实
- 轨道和章节切片是视图投影

因此不应因为参考资料强调时间线重要，就把时间线本身直接写成卷宗对象内部的一段私有结构。

更合理的口径是：

1. 时间事实优先落在 `events`、`relations.startAnchor/endAnchor` 等事实对象中。
2. 时间线展示优先通过 `views/chapter-slices.json`、后续轨道视图和派生聚合生成。

### 3. 把证据、完整度、异常结果写回对象真源

这些信息大部分属于：

1. 来源记录
2. 派生判断
3. 校验结果

它们不应被粗暴写回对象事实字段，否则会污染对象真源。

## 本轮吸收原则

### 对 `WBS 4.1`

重点吸收：

1. 更完整的卷宗产品定义
2. 更清晰的统一模块结构
3. 更明确的对象类型覆盖
4. 更完整的时间线 / 关系 / 事件 / 证据 / 分析 / 备注在卷宗中的位置
5. 更清晰的“哪些可编辑，哪些只是观察或派生”

### 对 `WBS 2.2`

重点吸收：

1. 卷宗统一观察模型与真实文件结构之间的映射
2. 关系记录增强口径
3. 时间锚点与章节投影的区分
4. 证据、完整度、异常结果的存储边界
5. 当前 `1.0.0` 强制规范与后续扩展规范的分层表达

## 本轮结论

参考资料确实显著强于我之前的 `WBS 4.1` 设计说明，我之前的问题主要有三点：

1. 只写到了“工作面存在”，没写清“卷宗为何成立”
2. 只写到了页面区块，没写清统一信息结构
3. 没有把数据真源、观察投影、派生分析三层彻底分开

本轮文档改写将围绕这三个问题修正，不会机械照抄参考资料，但会显式吸收其有效结构。
