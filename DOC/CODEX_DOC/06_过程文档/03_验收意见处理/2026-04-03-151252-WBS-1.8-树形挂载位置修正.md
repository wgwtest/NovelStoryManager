# WBS 1.8 树形挂载位置修正

## 问题来源

1. 用户反馈：`WBS 1.8` 在 GitHub Project 的树形结构中显示在最外层，挂载位置错误。

## 原因判断

1. `#39 WBS 1.8 BaseLab验证工具构建方案` 已被加入 GitHub Project。
2. 但它没有建立 GitHub 原生的父子 issue 关系：
   - `#39.parent = null`
   - `#9 WBS 1` 的 `subIssues` 中没有 `#39`
3. 结论：当前问题不是 Project 字段缺失，而是 issue 层级源数据缺失，导致 Project 将其显示为顶层节点。

## 修正决策

1. 使用 GitHub 原生 sub-issue 关系，把 `#39` 挂到 `#9` 下面。
2. 修正完成后复核：
   - `#39.parent` 应为 `#9`
   - `#9.subIssues` 应包含 `#39`
   - Project 树中 `#39` 不再作为最外层节点显示

## 不执行项

1. 不新增新的 `WBS` 节点。
2. 不修改 `WBS 1.8` 的设计方案正文。
3. 不进行任何代码改动。
