# WBS 1.7.2 原型路线重构分析

## 输入意见

用户明确指出上一版 `WBS 1.7.2` 五套稿件主要只是换配色，没有形成真正不同的设计路线。

## 意见合理性分析

该意见合理，原因如下：

1. 旧版 5 套稿件在左中右三栏总骨架上几乎完全同构。
2. 中区主工作面的信息组织方式没有发生本质变化。
3. 套系之间没有体现不同的工作流重心，只形成弱视觉差异。
4. 对于“剧情管理器”这类强工作台产品，真正的版本差异应体现在用户如何整理对象、如何推演关系、如何编排轨道、如何观察输出，而不是停留在换主题色。

## 外部参考检索结论

本轮重构以成熟工具的工作面组织方式为参考，而不是只参考视觉风格。

1. `Airtable Interface / Record Detail`
   - 参考点：结构化记录与详情联动
2. `Scrivener Corkboard`
   - 参考点：卡片化整理与卷宗组织
3. `Milanote / Obsidian Canvas`
   - 参考点：自由画布、归堆、拖拽组织
4. `Unreal Blueprint Editor / Obsidian Graph`
   - 参考点：节点依赖、端口语义、图结构推演
5. `Aeon Timeline / Miro Timeline / DaVinci Resolve`
   - 参考点：时间尺、多轨、事件编排篮
6. `Notion Timeline / Airtable Interface`
   - 参考点：总览面、联动视图和输出观察

## 重构策略

不保留旧版 5 套“轻度换肤”稿件，直接采用以下 5 条路线重做：

1. `卷宗案台型`
   - 核心结构：`卷宗目录`、`字段矩阵`、`记录详情抽屉`
2. `筹划墙型`
   - 核心结构：`筹划墙画布`、`灵感便签`、`素材归堆区`
3. `蓝图推演型`
   - 核心结构：`蓝图总线`、`推演节点`、`端口规则板`
4. `剪辑编排型`
   - 核心结构：`编排时间尺`、`轨道头矩阵`、`待排事件篮`
5. `总控指挥型`
   - 核心结构：`全局覆盖总览`、`联动监视窗`、`输出切片指挥板`

## 落地方式

1. 直接重写原型生成脚本，不在旧公共骨架上局部补丁。
2. 维持 `WBS 1.7.1` 冻结的中文命名、左侧对象库常驻和右侧检查器语义。
3. 中区主工作面按 5 条路线分别重构。
4. 每条路线继续输出总工作台、知识库、关系图、轨道、对象库与检查器 5 张图。

## 结论

本轮应把“多版本设计”从“多主题色”提升为“多工作流路线”，否则 `WBS 1.7.2` 无法为后续真实实现提供有效决策参考。
