# WBS 1.7.2 八套原型路线扩编与删减分析

## 输入意见

远端 `issue #43` 最新评论的有效含义可以整理为：

1. 每套方案中的 `5` 张图应当是 `5` 个不同页面，而不是同一概念模板重复铺图。
2. 当前稿件精度偏低，尤其 `蓝图推演型` 只停留在概念示意。
3. `卷宗案台型` 与 `剪辑编排型` 给出了有效启发。
4. `筹划墙型` 与 `总控指挥型` 价值不足，可以删除。
5. 需要在当前启发基础上继续扩展 `5` 套新模板。

## 意见合理性分析

这组意见合理，原因如下：

1. `WBS 1.7.2` 的目标不是证明“能画图”，而是提供不同工作台路线的比较样本。
2. 如果每套里的 `5` 张图没有页面职责分工，后续实现时就无法判断某条路线对三主视图是否真的成立。
3. 低价值路线继续保留，只会抬高人工评阅成本，降低路线池的有效密度。
4. 在已有有效启发上补充新路线，比在弱路线里反复抛光更符合当前阶段目标。

## 结构化设计解释

本轮按以下方式理解并执行评论：

1. 保留并深化 `卷宗案台型`、`蓝图推演型`、`剪辑编排型`。
2. 删除 `筹划墙型` 与 `总控指挥型`。
3. 新增 `档案阅览型`、`剧情弧矩阵型`、`场景调度型`、`双镜对读型`、`切片汇编型`。
4. 每套路线固定输出：
   - `总工作台总览`
   - `知识库页`
   - `关系图页`
   - `轨道页`
   - `对象库与检查器`
5. 五张图必须共享一套设计语言，但必须分别服务于五类页面职责。

## 路线池调整结果

### 保留并深化

1. `卷宗案台型`
   - 理由：最能体现对象建模、记录核验和对象库联动。
2. `蓝图推演型`
   - 理由：关系图和依赖推演方向仍然成立，问题在于旧稿表达过粗，而不是概念本身错误。
3. `剪辑编排型`
   - 理由：与多轨时间线、多空间线和观察切片的需求高度贴合。

### 删除

1. `筹划墙型`
   - 理由：更像灵感整理板，难以稳定承接三主视图和对象模型编辑。
2. `总控指挥型`
   - 理由：过于抽象，更像汇总面板，不适合作为稳定主工作面。

### 新增

1. `档案阅览型`
   - 目标：强化资料检索、正文阅览和脚注引用。
2. `剧情弧矩阵型`
   - 目标：强化剧情弧、人物、事件之间的覆盖观察。
3. `场景调度型`
   - 目标：突出地点层和空间切换，而不是只看时间轴。
4. `双镜对读型`
   - 目标：适配双线叙事、双视角对读和结构差异比较。
5. `切片汇编型`
   - 目标：强化观察切片和输出装配，但仍保持“章节不是建模前提”。

## 落地动作

1. 重写 `scripts/generate-wbs-1.7.2-prototypes.mjs` 的套系定义与页面构造逻辑。
2. 更新 `scripts/generate-wbs-1.7.2-prototypes.test.mjs`，把目录校验从 `5` 套提升到 `8` 套，并增加“每套内 5 张图是 5 个不同页面”的测试。
3. 重新生成 `.pen` 原稿。
4. 重新导出 `.png` 图片。
5. 同步更新设计说明、研制计划、执行说明和 README。

## 结论

本轮不再把“多版本原型”理解成多种配色或多种装饰，而把它收敛为“多种主工作流路线”。扩编到 `8` 套以后，`WBS 1.7.2` 才重新具备比较和筛选价值。
