# WBS 1.8.2 拖拽与动态连线验收意见处理

## 评论来源

1. GitHub issue：`#41 WBS 1.8.2 WBS 3.1实验迁移与入口解耦`
2. 评论编号：`4182637421`
3. 评论时间：`2026-04-03T09:03:24Z`
4. 评论原文：
   - 当前的背景版，三个都有同样的问题：一拖拽就放缩很大，飘得很远；我没有看到节点上动态连线的能力

## 需求理解

1. 用户不是在要求新的抽象分析，而是在指出当前 `BaseLab` 实验页存在两个可直接观察的问题。
2. 第一个问题是三种基座在拖拽交互上都存在异常，表现为拖一下就出现明显失控位移。
3. 第二个问题是当前实验页没有体现“从节点端口动态拖出连线”的能力，因此还不足以支撑关系图交互方案判断。

## 代码现实核对

1. `apps/base-lab/src/components/BaseSelectionLab.tsx` 中三种基座共用 `useInteractiveStageState()`。
2. 其中画布平移在 `handlePointerMove()` 内使用：
   - `panCanvasViewport(current, { deltaX: event.clientX - activeDrag.startX - activeDrag.originOffsetX, ... })`
3. 该写法把“当前指针位移”和“初始 offset”混在一起，并在每次 `pointermove` 中继续叠加到 `current`，会导致偏移计算漂移。
4. 当前节点只展示了静态预置边：
   - `labEdges` 为常量数组
   - 节点端口没有单独的拖出连线手势
   - 页面也没有“连线预览”或“拖到目标端口后新建边”的实现

## 判断

1. 该评论合理，且属于当前 `WBS 3.1` 验证页的有效缺口，不是未来节点才能处理的事项。
2. 拖拽异常属于当前实现缺陷，应在本轮修正。
3. 动态连线能力虽然不等于完整关系图编辑器，但作为“基座选型验证页”，至少应补齐：
   - 输出端口拖出连线预览
   - 拖到输入端口时完成连线
   - 三种基座共享同一交互任务
4. 不需要在本轮扩成正式关系图工作台；只要补到足以验证基座差异即可。

## 本轮执行决定

1. 先补失败测试，覆盖：
   - 视口平移按真实指针位移更新，不再出现偏移漂移
   - `BaseLab` 中能从端口拖出动态连线并创建新边
2. 再做最小代码实现：
   - 修正 `useInteractiveStageState()` 的平移计算
   - 将边集合纳入 stage state
   - 补充端口命中、连线预览和落点连接
3. 同轮更新：
   - `WBS 1.8.2` / `WBS 3.1` 相关计划或验收条目
   - 自测报告
   - 交接文档
   - GitHub issue 评论同步
