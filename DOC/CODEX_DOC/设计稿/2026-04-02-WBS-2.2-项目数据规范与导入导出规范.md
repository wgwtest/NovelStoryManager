# WBS 2.2 项目数据规范与导入导出规范

## 目标

1. 固化单项目文件存储与对外导入导出之间的稳定映射。
2. 让外部智能体只要按规范生成结构化数据，就能被本工具校验和载入。
3. 保持“对象库是真源、视图状态独立、AI 理解外置”这三条前置约束不被破坏。

## 作用边界

1. 本规范只定义结构化数据长什么样、如何校验、如何进出系统。
2. 本规范不负责正文理解、章节自动解析、关系自动推断。
3. 本规范不改变内部真源：内部真源仍然是项目目录下的多文件对象库。

## 内部项目文件规范

### 目录结构

1. 项目根目录必须包含：
   - `manifest.json`
   - `schema-version.json`
   - `objects/`
   - `views/`
2. `objects/` 下按对象类型分文件存储：
   - `characters.json`
   - `factions.json`
   - `locations.json`
   - `items.json`
   - `realm-systems.json`
   - `events.json`
   - `relations.json`
   - `clues.json`
   - `arcs.json`
3. `views/` 下按视图状态分文件存储：
   - `graph-layouts.json`
   - `track-presets.json`
   - `saved-filters.json`

### 真源约束

1. 对象事实只允许写入 `objects/*.json`。
2. 视图编排和布局只允许写入 `views/*.json`。
3. 章节文本、正文理解结果和 AI 推断结果不属于本节点输入。

## 对外导入导出规范

### 项目级 bundle

项目级 bundle 用于完整导入或完整导出一个小说项目，建议文件名为 `project-bundle.json`。

建议结构：

```json
{
  "bundleType": "project-bundle",
  "bundleVersion": "1.0.0",
  "exportMeta": {
    "sourceProjectId": "sample-novel",
    "exportedAt": "2026-04-02T00:00:00Z"
  },
  "project": {
    "manifest": {},
    "schemaVersion": {},
    "objects": {},
    "views": {}
  }
}
```

约束：

1. `bundleType` 必须为 `project-bundle`。
2. `bundleVersion` 必须命中当前受支持版本。
3. `project.manifest`、`project.schemaVersion`、`project.objects`、`project.views` 均为必填。
4. `project.objects` 必须满足内部对象集合 schema。
5. `project.views` 必须满足内部视图集合 schema。

### 对象级 bundle

对象级 bundle 用于结构化增量导入或按对象范围导出，建议文件名为 `object-batch.json`。

建议结构：

```json
{
  "bundleType": "object-batch",
  "bundleVersion": "1.0.0",
  "exportMeta": {
    "sourceProjectId": "sample-novel",
    "exportedAt": "2026-04-02T00:00:00Z"
  },
  "scope": {
    "targetProjectId": "sample-novel",
    "objectTypes": ["characters", "relations"]
  },
  "objects": {
    "characters": [],
    "relations": []
  }
}
```

约束：

1. `bundleType` 必须为 `object-batch`。
2. `scope.targetProjectId` 必须存在。
3. `scope.objectTypes` 必须与 `objects` 内的集合类型一致。
4. 对象级 bundle 默认不携带 `views`，避免把布局状态混入对象事实。
5. 导入时若发现引用对象既不在当前 batch 中，也不在目标项目中，必须失败，不允许静默跳过。

## 校验规则

1. 根字段缺失必须失败。
2. `bundleType` 非法必须失败。
3. `bundleVersion` 不受支持必须失败。
4. 同一对象集合中 `id` 重复必须失败。
5. 关系、事件、线索、剧情弧中的对象引用无法解析时必须失败。
6. 项目级 bundle 导入后导出的结构应与导入前保持语义一致。
7. 对象级 bundle 导入时只允许改动目标对象集合，不得污染未声明对象类型。

## 版本规则

1. `schema-version.json` 继续表达项目内部 schema 版本。
2. `bundleVersion` 单独表达导入导出包规范版本。
3. `schemaVersion` 与 `bundleVersion` 任一方不受支持时，都必须快速失败。
4. 本节点只定义 `1.0.0` 初版，不在本轮实现自动迁移。

## 失败处理规则

1. 校验失败时必须返回明确错误，不允许自动修补数据。
2. 对象级导入遇到坏引用时必须整体失败，不允许部分成功。
3. 版本不兼容时必须返回版本错误，而不是落入通用解析错误。

## 非目标

1. 不在本节点实现章节维度建模。
2. 不在本节点实现关系图或轨道视图的真实交互。
3. 不在本节点内置任何 AI 文本抽取或语义理解能力。
