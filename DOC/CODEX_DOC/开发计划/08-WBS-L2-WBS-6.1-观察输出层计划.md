# WBS 6.1 观察输出层计划

## 目标

在不新增主工作台标签、不改写对象事实文件的前提下，于 Tracks 工作台内交付首个可用的观察输出层，让用户按时间、人物、地点、剧情线四个维度查看派生切片，并为未来章节观察维度保留接入位置。

## 本节点必须交付的成果物

1. 观察切片派生逻辑：
   - `apps/web/src/lib/project-observation.ts`
2. Tracks 内观察面板：
   - `apps/web/src/components/TracksView.tsx`
3. 回归测试：
   - `apps/web/src/App.test.tsx`
4. 样式补齐：
   - `apps/web/src/styles.css`

## 实施分块

### A. 派生观察接口

1. 基于事件对象派生时间切片。
2. 基于事件引用派生人物、地点、剧情线切片。
3. 明确观察结果只读，不回写对象事实。

### B. Tracks 内观察面板

1. 在 Tracks 工具栏增加观察维度切换。
2. 在 Tracks 工作区下方展示观察切片面板。
3. 支持从观察切片打开对应对象或事件。

### C. 章节维度预留位

1. 在观察面板中提供章节维度占位说明。
2. 明确当前仅预留承接口，不要求存在章节模型。

### D. 测试与回归

1. 增加四类切片渲染测试。
2. 增加章节维度预留位测试。
3. 补跑根工作区类型检查、测试和构建。

## 强制测试矩阵

1. 时间切片：
   - Tracks 中可看到 `卷一-早期` 切片，并列出 `试炼谷夺令`。
2. 人物切片：
   - 切换到 `character` 后，可看到 `苏玄` 切片。
3. 地点切片：
   - 切换到 `location` 后，可看到 `青云峰` 切片。
4. 剧情线切片：
   - 切换到 `arc` 后，可看到 `隐火灵脉主线` 切片。
5. 章节承接：
   - 面板存在 `Chapter Dimension` 预留位，并提示未来结构化章节切片接入。
6. 工作区回归：
   - `npm run test --workspace @novelstory/web`
   - `npm run typecheck`
   - `npm run test`
   - `npm run build`
   - 上述命令全部退出码为 `0`。

## 非目标

1. 不在本节点实现章节文本建模。
2. 不在本节点实现章节导入或正文解析。
3. 不在本节点新增第四个主工作台标签。
4. 不在本节点实现观察结果持久化。
