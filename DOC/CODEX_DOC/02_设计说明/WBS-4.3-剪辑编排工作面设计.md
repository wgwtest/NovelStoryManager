# WBS 4.3 剪辑编排工作面设计

## 工作面定位

`剪辑编排` 是多线叙事编排工作面。

之所以使用“剪辑编排”这个名字，而不是简单叫“轨道视图”，是因为它对应的是一种明确的产品心智：

1. 把故事素材放入多个轨道
2. 像剪辑软件一样观察顺序、并列与覆盖
3. 在时间线、空间线、剧情线之间切换或并列观察

它不是一个普通的列表页，而是“多轨编排台”。

## 解决的问题

剪辑编排工作面主要解决以下问题：

1. 事件顺序、并列关系和节奏变化难以整体观察
2. 同一对象可能同时参与时间线、空间线和剧情线，缺少统一编排面
3. 章节和切片需要从编排结果中提取，而不是反过来约束建模

因此，章节在这里是观察结果，而不是建模前提。

## 与其他工作面的边界

### 1. 剪辑编排负责什么

1. 多轨组织
2. 片段入轨与换轨
3. 顺序调整
4. 时间锚点和切片观察
5. 章节输出前的编排观察

### 2. 剪辑编排不负责什么

1. 不负责对象事实核对
2. 不负责节点逻辑推演
3. 不负责空间拓扑与舞台关系

## 数据驱动关系

### 1. 主要输入数据

剪辑编排读取共享对象真源，尤其依赖：

1. `events.json`
2. `characters.json`
3. `locations.json`
4. `arcs.json`
5. `relations.json`

### 2. 视图状态输入

剪辑编排同时依赖编排类视图状态，例如：

1. `views/track-presets.json`
2. `views/chapter-slices.json`
3. 当前视口位置
4. 当前选中轨道与片段

### 3. 输出动作

剪辑编排会产生：

1. 轨道预设变更
2. 片段布局变化
3. 章节切片结果
4. 局部观察状态

其中前两类和后两类都属于视图与观察结果，不应把轨道摆放结果回写成对象事实。

## 页面结构

### 1. 左侧对象库

对象库在这里主要承担素材池入口：

1. 角色
2. 事件
3. 地点
4. 剧情弧
5. 其他可被拖入轨道的对象

### 2. 中央编排区

中央主区至少包含：

1. 时间尺
2. 轨道头
3. 轨道体
4. 片段块
5. 多线分组区

### 3. 辅助片段区

编排页可提供片段篮、预设区或切片区，用于承载：

1. 尚未入轨的片段
2. 常用编排模板
3. 章节切片结果

### 4. 右侧检查器

检查器用于查看当前片段块、轨道或切片的属性摘要。

## 关键交互

1. 从对象库或片段篮拖入轨道
2. 在轨道间换轨
3. 在同轨内重排顺序
4. 视口平移与缩放
5. 以时间、空间或剧情线分组观察
6. 从编排结果生成章节切片

## 设计约束

1. 必须保留明显的多轨心智，不允许退化成普通表格。
2. 必须明确区分“未入轨”“预放置”“已入轨”“非法落点”。
3. 章节切片是编排结果的观察输出，不得反向成为对象建模的必填前置。
4. 剪辑编排只共享对象真源与公共协议，不共享其他工作面的中央实现。
