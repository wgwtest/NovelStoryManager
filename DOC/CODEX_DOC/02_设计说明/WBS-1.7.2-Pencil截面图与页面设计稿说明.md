# WBS 1.7.2 Pencil截面图与页面设计稿说明

## 目标

基于 `WBS 1.7.1` 已明确的页面语义和布局规则，直接采用 `Pencil Project + Codex` 产出页面设计稿与截面图，而不是把 `Pencil` 当成“最后补一笔”的工具。

## 执行方式

1. `Codex` 负责将口述意见转写成结构化设计清单。
2. `Pencil Project` 负责直接承接这些清单，输出静态页面设计稿。
3. 当前不引入“外部自动生成草稿 -> `Pencil` 精修”的中间环节。

## Pencil 的角色定位

1. 直接承担本轮页面设计稿产出。
2. 重点负责总工作台、对象库、知识库主工作区以及局部高信息密度区域的页面设计。
3. 对强动态交互本身，只表达静态状态和布局，不代替后续真实交互实现。

## 路线重构结论

用户已明确指出上一版 5 套稿件只形成“同一骨架换配色”的弱差异，这个判断是合理的。

本轮因此不再把“版本差异”理解成配色或卡片细节差异，而是要求 5 套方案必须至少在以下方面形成明确区别：

1. 中区主工作面的核心隐喻不同
2. 知识库、关系图、轨道三视图在套系中的主次关系不同
3. 中区的局部结构名称和工作流动作不同
4. 用户处理对象、关系、轨道和切片的方式不同

## 外部参考转译

本轮设计路线不从“好看界面”出发，而从成熟工具的工作面组织方式出发。

1. `Airtable Interface / Record Detail`
   - 参考要点：列表、字段矩阵、记录详情联动
   - 转译目标：`卷宗案台型`
2. `Scrivener Corkboard`
   - 参考要点：卡片化组织和卷宗式浏览
   - 转译目标：`卷宗案台型`
3. `Milanote / Obsidian Canvas`
   - 参考要点：自由拖拽、归堆、无限画布组织
   - 转译目标：`筹划墙型`
4. `Unreal Blueprint Editor / Obsidian Graph`
   - 参考要点：节点推演、端口语义、依赖流向
   - 转译目标：`蓝图推演型`
5. `Aeon Timeline / Miro Timeline / DaVinci Resolve`
   - 参考要点：时间尺、多轨、待排素材篮
   - 转译目标：`剪辑编排型`
6. `Notion Timeline / Airtable Interface`
   - 参考要点：多视图联动、总览面、记录到输出联动
   - 转译目标：`总控指挥型`

## 首批设计稿包

本轮采用“同一语义基线下的 5 套结构路线并行比较”方式推进，而不是“同一后台壳子做 5 次轻微换肤”。

五套方案方向固定为：

1. `卷宗案台型`
2. `筹划墙型`
3. `蓝图推演型`
4. `剪辑编排型`
5. `总控指挥型`

### 1. 卷宗案台型

1. 核心隐喻：对象卷宗与字段审校案台
2. 中区关键结构：
   - `卷宗目录`
   - `字段矩阵`
   - `记录详情抽屉`
3. 主价值：让用户把建模对象当作“待整理卷宗”来处理

### 2. 筹划墙型

1. 核心隐喻：拖拽筹划墙与素材归堆区
2. 中区关键结构：
   - `筹划墙画布`
   - `灵感便签`
   - `素材归堆区`
3. 主价值：让用户先搭剧情组织关系，再回写对象模型

### 3. 蓝图推演型

1. 核心隐喻：蓝图总线与节点推演
2. 中区关键结构：
   - `蓝图总线`
   - `推演节点`
   - `端口规则板`
3. 主价值：把对象、关系、事件和切片的流向表达成可推演结构

### 4. 剪辑编排型

1. 核心隐喻：剪辑时间尺与多轨编排
2. 中区关键结构：
   - `编排时间尺`
   - `轨道头矩阵`
   - `待排事件篮`
3. 主价值：让时间线、空间线、观察切片像剪辑软件一样组织

### 5. 总控指挥型

1. 核心隐喻：跨视图总控台
2. 中区关键结构：
   - `全局覆盖总览`
   - `联动监视窗`
   - `输出切片指挥板`
3. 主价值：把知识库、关系图、轨道统一放到一个观察与调度平面中

### 1. 总工作台总览图

1. 白色中性工作台基调
2. 小型产品标题
3. 当前项目名次级展示
4. 中文 tab 的紧凑横排
5. 左中右三列清晰分工

### 2. 左侧对象库设计稿

1. 搜索框优先
2. 分类切换次之
3. 列表区高密度展示
4. 覆盖九类对象的中文分类标签

### 3. 知识库视图设计稿

1. 说明“知识库”不是章节页，而是结构化真值编辑页
2. 中列展示结构化列表或主编辑区域
3. 右列检查器展示选中对象的辅助详情与引用跳转

### 4. 关系图局部截面图

1. Unreal 蓝图风格节点
2. 清晰输入端口、输出端口与参数区
3. 端口吸附与连线提示的静态表达

### 5. 轨道局部截面图

1. 时间尺
2. 轨道头
3. 事件块与观察切片
4. 多轨并排与吸附位置提示

## 视觉约束

1. 主基调为白色与中性灰，不再使用暖黄底。
2. 重点强调信息密度与编辑器气质，而不是展示性海报风格。
3. 全页面统一中文标签。
4. 样例项目名 `青云劫火录` 只作为项目上下文，不作为主视觉标题。

## 输出形式

1. 原型稿统一落盘到 `DOC/CODEX_DOC/04_研发文档/03_原型图/WBS-1.7.2/`
2. 每套方案单独建立目录，目录内至少包含：
   - `Pencil` 原稿文件
   - 导出图片或 PDF
   - 一份简短设计说明
3. 每张设计稿都要明确标注：
   - 所属视图
   - 所属区域
   - 设计目的

## 约束

1. 所有设计稿必须复用 `WBS 1.7.1` 中的中文命名、栏目职责和布局顺序。
2. 不把 `Pencil` 原稿直接当成代码实现说明。
3. 如果某个布局改动无法解释其语义价值，就不应写入设计稿。
4. 不允许再出现“仅换配色、不换骨架”的伪多版本设计。
