# WBS 1.7.2 Pencil截面图与页面设计稿说明

## 目标

基于 `WBS 1.7.1` 已冻结的页面语义和栏目职责，直接使用 `Pencil Project + Codex` 产出可用于路线判断的静态设计稿。本轮不再追求“多套路线横向比较”，而是把页面结构直接收束为 `4` 个核心工作面，每个工作面只保留 `1` 张讲清楚主结构的核心图。

## 本轮评阅解释

远端 `issue #43` 的最新评论 `4188222166` 可以结构化理解为以下几点：

1. `档案阅览型`、`剧情弧矩阵型`、`双镜对读型`、`切片汇编型` 价值不足，当前轮次全部移除。
2. 只保留 `卷宗`、`蓝图推演`、`剪辑编排`、`场景调度` 四个工作面方向。
3. 不再需要“每套 `5` 张图”的表达方式，每个方向只保留 `1` 张核心图。
4. 原先 `知识库 / 关系图 / 轨道` 的页面拆分，不再作为本轮原型的主导航结构。
5. 新一轮原型不只是细化信息密度，还要直接调整产品页面结构本身。
6. `字段矩阵`、`记录详情抽屉`、`卷宗审校进度` 这类术语过虚，必须改成用户一眼能理解的工作面表达。

这个判断合理，因为当前节点的价值已经从“继续扩路线”转成“把真正保留下来的页面方向讲透”。如果还保留八套路线或五页拆分，只会继续稀释设计判断。

## 页面结构调整

本轮产品结构改为：

1. 左侧固定 `对象库`
2. 中央主工作面切换为：
   - `卷宗`
   - `蓝图推演`
   - `剪辑编排`
   - `场景调度`
3. 右侧固定 `检查器`

因此，本轮原型不再把中区理解成“知识库页 / 关系图页 / 轨道页”的旧三分法，而是改成四种不同的建模工作角度。

## 四个核心工作面定义

### 1. 卷宗工作面

1. 核心问题：如何把对象建模做成可核验、可追踪、可回看的卷宗整理工作
2. 主工作区关键词：
   - `卷宗目录`
   - `卷宗摘要`
   - `对象审校`
3. 设计重点：
   - 左侧对象切换要足够直接
   - 中区要突出对象当前状态、关联对象和待补齐信息
   - 右侧检查器负责当前条目字段与说明

### 2. 蓝图推演工作面

1. 核心问题：如何把对象、事件、依赖和推导关系组织成可观察的推演结构
2. 主工作区关键词：
   - `蓝图总线`
   - `推演节点`
   - `依赖回路`
3. 设计重点：
   - 不把图形画成抽象装饰
   - 要让节点层次、依赖方向和回路检查一眼可见
   - 右侧检查器承担规则解释和当前节点参数

### 3. 剪辑编排工作面

1. 核心问题：如何把时间线、观察切片和并行剧情组织成真正可编排的多层工作区
2. 主工作区关键词：
   - `编排时间尺`
   - `片段篮`
   - `编排层`
3. 设计重点：
   - 中区必须接近剪辑软件心智
   - 片段进入时间尺、在不同编排层之间排布的关系要清楚
   - 不把它退化成普通表格或清单

### 4. 场景调度工作面

1. 核心问题：如何把地点、空间跳转和场景流转组织成空间调度视角
2. 主工作区关键词：
   - `场景舞台图`
   - `地点调度`
   - `空间切换`
3. 设计重点：
   - 强调空间观察，而不是只强调时间排序
   - 让地点切换、舞台占用和场景流转具有直接可读性
   - 保持和对象库、检查器的联动

## 执行方式

1. `Codex` 负责把评阅意见转成结构化设计要求，并同步到设计说明、研制计划、自测报告和生成脚本。
2. `Pencil Project` 负责承接这些要求，产出统一格式的 `.pen` 原稿。
3. `Chrome Headless` 导出链路负责根据同一套工作面语义生成稳定可评审的 `.png` 导出图，避免 `Pencil CLI` 在复杂嵌套布局下出现空白导出。
4. 当前原型稿只保留 `4` 组工作面文件，统一平铺在 `WBS-1.7.2/` 根目录下，每组只产出：
   - `1` 个 `.pen`
   - `1` 个 `.png`
   - `1` 份 `-方案说明.md`

## 视觉与语义约束

1. 产品标题统一使用 `知识编辑器`。
2. 样例项目名 `青云劫火录` 只作为当前项目上下文，不作为主视觉标题。
3. 主工作面统一使用白色中性工作台，不使用暖黄底。
4. 左侧对象库顺序统一为：搜索框、分类切换、统计动作、对象列表。
5. 对象库统一覆盖九类对象：人物、势力、地点、物品、境界体系、事件、关系、线索、剧情弧。
6. 所有标签、按钮、说明、区域标题统一中文化。
7. 顶部主导航统一为：`卷宗`、`蓝图推演`、`剪辑编排`、`场景调度`。

## 约束

1. 所有设计稿必须复用 `WBS 1.7.1` 中已经澄清的中文命名、栏目职责和布局顺序。
2. 不把 `Pencil` 原稿直接当成代码实现说明。
3. 如果某个布局无法说清其语义价值，就不应写入设计稿。
4. 不再追求“多方案数量”，而是追求“保留下来的四个工作面是否讲清楚”。
