# WBS 1.7.2 Pencil截面图与页面设计稿说明

## 目标

基于 `WBS 1.7.1` 已冻结的页面语义和栏目职责，直接使用 `Pencil Project + Codex` 产出高信息密度的静态设计稿，用于比较不同工作台路线，而不是把 `Pencil` 当成后置补图工具。

## 本轮评阅解释

远端 `issue #43` 的最新评论可以结构化理解为以下几点：

1. 每套方案中的 `5` 张图必须是 `5` 个不同页面，而不是同一母版换标签。
2. 页面精度要明显提升，不能只用几个概念方块占位。
3. 当前稿件里 `卷宗案台型` 与 `剪辑编排型` 有价值，`蓝图推演型` 的概念仍成立，但原稿表达过粗。
4. `筹划墙型` 与 `总控指挥型` 的价值不足，可以删除。
5. 需要在当前启发基础上继续补 `5` 套新模板，而不是停留在旧五套内部修补。

这个判断合理，因为 `WBS 1.7.2` 的核心价值本来就是为后续真实实现提供路线判断依据。如果同一套骨架只换主题色，设计稿就没有比较价值。

## 执行方式

1. `Codex` 负责把口述评阅转成结构化设计要求，并固化到设计说明、计划文档、自测报告和生成脚本。
2. `Pencil Project` 负责承接这些要求，产出统一格式的 `.pen` 原稿和 `.png` 导出图。
3. 当前不引入“自动草图工具先出图，再回到 Pencil 精修”的中间层。

## 页面拆分原则

每套方案固定输出以下 `5` 个页面，但它们必须是同一设计语言下的异构页面：

1. `总工作台总览`
   - 用来说明该套路线的总工作面组织方式和三主视图入口。
2. `知识库页`
   - 用来说明结构化真值编辑如何落在这套路线里。
3. `关系图页`
   - 用来说明对象、事件、关系、线索如何在这套路线里表达。
4. `轨道页`
   - 用来说明多时间线、多空间线和观察切片如何编排。
5. `对象库与检查器`
   - 用来说明左侧对象导航和右侧详情联动，而不是重复中区布局。

因此，“同一元素在 5 张图里机械平铺”不再允许。每张图必须服务于对应页面任务。

## 路线调整结论

本轮不再维持旧版 5 套结构，而改为 `3` 条保留深化路线加 `5` 条新增路线，总计 `8` 套：

1. 保留并深化：
   - `卷宗案台型`
   - `蓝图推演型`
   - `剪辑编排型`
2. 新增补充：
   - `档案阅览型`
   - `剧情弧矩阵型`
   - `场景调度型`
   - `双镜对读型`
   - `切片汇编型`
3. 删除路线：
   - `筹划墙型`
   - `总控指挥型`

## 八套路线定义

### 1. 卷宗案台型

1. 核心隐喻：对象卷宗与字段审校案台
2. 核心结构：`卷宗目录`、`字段矩阵`、`记录详情抽屉`
3. 主价值：把建模对象当作待整理、待核验的记录卷宗来处理

### 2. 档案阅览型

1. 核心隐喻：资料索引架与正文阅览页
2. 核心结构：`档案索引架`、`引文页框`、`参考脚注列`
3. 主价值：强调“索引 -> 阅览 -> 引用”的资料工作流

### 3. 蓝图推演型

1. 核心隐喻：节点总线与依赖推演台
2. 核心结构：`蓝图总线`、`推演节点`、`端口规则板`
3. 主价值：把对象、事件、关系和切片放进可推演的图结构里

### 4. 剧情弧矩阵型

1. 核心隐喻：剧情弧观察矩阵
2. 核心结构：`剧情弧矩阵`、`角色交叉轴`、`事件覆盖格`
3. 主价值：把剧情弧、人物、事件的覆盖关系压到同一观察平面中

### 5. 剪辑编排型

1. 核心隐喻：剪辑时间尺与多轨编排台
2. 核心结构：`编排时间尺`、`轨道头矩阵`、`待排事件篮`
3. 主价值：把多时间线、多空间线和观察切片当作剪辑素材来编排

### 6. 场景调度型

1. 核心隐喻：场景舞台与地点调度盘
2. 核心结构：`场景舞台图`、`地点调度层`、`空间切换条`
3. 主价值：突出地点层、主舞台和空间跳切，而不是只从时间轴看故事

### 7. 双镜对读型

1. 核心隐喻：双视角对照工作台
2. 核心结构：`双镜对读台`、`左右对照窗`、`差异摘要带`
3. 主价值：适合比较双线叙事、双角色视角或双组关系结构

### 8. 切片汇编型

1. 核心隐喻：观察切片装配台
2. 核心结构：`切片汇编台`、`片段编排列`、`输出装配栏`
3. 主价值：把对象模型投影成可交付的观察切片，但仍不把章节当作建模前提

## 视觉与语义约束

1. 产品标题统一使用 `知识编辑器`。
2. 样例项目名 `青云劫火录` 只作为当前项目上下文，不作为主视觉标题。
3. 主工作面统一使用白色中性工作台，不使用暖黄底。
4. 三主视图统一使用中文紧凑 tab：`知识库`、`关系图`、`轨道`。
5. 左侧对象库顺序统一为：搜索框、分类切换、统计动作、对象列表。
6. 对象库统一覆盖九类对象：人物、势力、地点、物品、境界体系、事件、关系、线索、剧情弧。
7. 所有标签、按钮、说明、区域标题统一中文化。

## 输出形式

1. 原型稿统一落盘到 `DOC/CODEX_DOC/04_研发文档/03_原型图/WBS-1.7.2/`
2. 每套方案单独建立目录，目录内至少包含：
   - `5` 个 `.pen` 原稿
   - `5` 个对应 `.png` 导出图
   - `1` 份 `方案说明.md`
3. 设计稿必须同时服务于：
   - 远端人工评阅
   - 后续前端实现路线选择
   - 视图语义冻结前的快速比较

## 约束

1. 所有设计稿必须复用 `WBS 1.7.1` 中已经澄清的中文命名、栏目职责和布局顺序。
2. 不把 `Pencil` 原稿直接当成代码实现说明。
3. 如果某个布局无法说清其语义价值，就不应写入设计稿。
4. 不允许再出现“仅换配色、不换骨架”的伪多版本设计。
