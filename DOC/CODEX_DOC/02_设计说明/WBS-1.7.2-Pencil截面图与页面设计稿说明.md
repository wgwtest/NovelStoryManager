# WBS 1.7.2 Pencil截面图与页面设计稿说明

## 目标

基于 `WBS 1.7.1` 已明确的页面语义和布局规则，直接采用 `Pencil Project + Codex` 产出页面设计稿与截面图，而不是把 `Pencil` 当成“最后补一笔”的工具。

## 执行方式

1. `Codex` 负责将口述意见转写成结构化设计清单。
2. `Pencil Project` 负责直接承接这些清单，输出静态页面设计稿。
3. 当前不引入“外部自动生成草稿 -> `Pencil` 精修”的中间环节。

## Pencil 的角色定位

1. 直接承担本轮页面设计稿产出。
2. 重点负责总工作台、对象库、知识库主工作区以及局部高信息密度区域的页面设计。
3. 对强动态交互本身，只表达静态状态和布局，不代替后续真实交互实现。

## 首批设计稿包

### 1. 总工作台总览图

1. 白色中性工作台基调
2. 小型产品标题
3. 当前项目名次级展示
4. 中文 tab 的紧凑横排
5. 左中右三列清晰分工

### 2. 左侧对象库设计稿

1. 搜索框优先
2. 分类切换次之
3. 列表区高密度展示
4. 覆盖九类对象的中文分类标签

### 3. 知识库视图设计稿

1. 说明“知识库”不是章节页，而是结构化真值编辑页
2. 中列展示结构化列表或主编辑区域
3. 右列检查器展示选中对象的辅助详情与引用跳转

### 4. 关系图局部截面图

1. Unreal 蓝图风格节点
2. 清晰输入端口、输出端口与参数区
3. 端口吸附与连线提示的静态表达

### 5. 轨道局部截面图

1. 时间尺
2. 轨道头
3. 事件块与观察切片
4. 多轨并排与吸附位置提示

## 视觉约束

1. 主基调为白色与中性灰，不再使用暖黄底。
2. 重点强调信息密度与编辑器气质，而不是展示性海报风格。
3. 全页面统一中文标签。
4. 样例项目名 `青云劫火录` 只作为项目上下文，不作为主视觉标题。

## 输出形式

1. `Pencil` 原稿文件
2. 供评阅使用的导出图片或 PDF
3. 每张设计稿都要明确标注：
   - 所属视图
   - 所属区域
   - 设计目的

## 约束

1. 所有设计稿必须复用 `WBS 1.7.1` 中的中文命名、栏目职责和布局顺序。
2. 不把 `Pencil` 原稿直接当成代码实现说明。
3. 如果某个布局改动无法解释其语义价值，就不应写入设计稿。
