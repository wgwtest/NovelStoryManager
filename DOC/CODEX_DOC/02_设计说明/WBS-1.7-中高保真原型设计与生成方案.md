# WBS 1.7 中高保真原型设计与生成方案

## 说明

本节点沿用既有 `WBS 1.7` 编号与文件名，以保持历史引用稳定。
但在 `2026-04-03` 晚间评论后，当前执行口径已经调整为“`Pencil Project + Codex` 直接协同设计”，不再把“自动生成多页面草稿”作为主路线前提。

## 目标

在现有低保真原型与交互验证基础上，为主工作台建立一套可直接执行的中高保真设计路线，使后续页面设计先把语义、栏目职责和视觉密度讲清楚，再进入具体设计稿产出。

## 评阅触发的路线调整

1. 用户明确否定“外部自动生成工具先出草稿，再由 `Pencil` 精修”的执行方式。
2. 用户明确要求直接以 `Pencil Project` 配合 `Codex` 进行设计。
3. 用户当前评论是对着现有原型页口述得出，因此本节点必须先完成“口述意见转写与结构化理解”，再开始设计。
4. 当前页面存在的主要问题不是单一配色问题，而是：
   - 产品命名与项目标题混淆
   - 视图语义未讲明白
   - 英文化标签过多
   - 布局密度松散
   - 左中右三列职责表达不清

## 当前主路线

### 路线定义

1. `Codex` 负责把口述评论转成明确设计约束、语义定义、栏目职责说明和设计清单。
2. `Pencil Project` 负责直接产出静态页面设计稿和截面图。
3. 当前不再把 `Uizard / Visily / Figma AI` 等自动生成工具作为必经步骤。

### 路线原因

1. 这与用户的明确意图一致。
2. 本项目强调文件化、可导入导出和本地资产沉淀，`Pencil` 更贴合当前协作方式。
3. 当前真正缺的不是“多出几张草稿”，而是先把页面为什么这样分栏、这些栏目到底表达什么讲清楚。

## 当前页面的强约束

1. 主色调改为白色中性工作台，不再使用偏黄底色。
2. 顶部主标题不再突出小说题名，产品名采用小型通用标题，例如 `知识编辑器`。
3. 当前项目标题只作为轻量项目上下文出现，例如副标题、面包屑或状态区补充信息。
4. `Knowledge / Graph / Tracks` 改为中文紧凑 tab，不再使用大块英文占位。
5. 页内按钮、栏目标题、注释、占位文案统一改成中文。
6. 左侧对象库改成“搜索框优先、分类其次、列表最后”的顺序。
7. 左侧对象库分类至少覆盖：
   - 人物
   - 势力
   - 地点
   - 物品
   - 境界体系
   - 事件
   - 关系
   - 线索
   - 剧情弧
8. 三主视图与右侧检查器的职责必须先写明白，再进入设计稿阶段。

## 三主视图的执行语义

### 知识库视图

1. 这是对“对象真值”的结构化浏览与编辑页。
2. 它不是章节文本页，也不是关系图页。
3. 它的核心任务是按对象分类查看、筛选、定位和编辑人物、势力、地点、物品、事件等结构化数据。

### 关系图视图

1. 这是对象关系与依赖结构的可视化编辑页。
2. 它重点处理节点创建、连线、端口、锚点、可连接性和画布布局。
3. 节点风格继续沿 Unreal 蓝图式节点方向收敛。

### 轨道视图

1. 这是时间线、空间线和观察切片的多轨编排页。
2. 它不是事实真源，而是对对象与事件的观察性编排。
3. 它重点处理轨道、时间尺、吸附、排序、切片观察位和章节覆盖关系。

## 左中右三列职责

1. 左列是 `对象库`：
   - 承担搜索、分类切换、对象定位、快速创建入口
   - 不承担关系可视化，也不承担章节编排
2. 中列是当前主视图工作区：
   - `知识库` 负责结构化浏览与编辑
   - `关系图` 负责图式编辑
   - `轨道` 负责多轨编排
3. 右列是 `检查器`：
   - 承担当前选择对象的详细字段、引用关系、状态说明和快速跳转
   - 必须是“有解释的辅助详情区”，不是任意堆字段的杂项栏

## WBS 拆分补充

1. `WBS 1.7` 保持为父节点，负责冻结：
   - 直接设计路线
   - 页面语义约束
   - 中文化与紧凑布局原则
   - 设计稿输出包边界
2. `WBS 1.7.1` 负责把页面语义、栏目职责、中文命名和布局规则写清楚。
3. `WBS 1.7.2` 负责基于 `WBS 1.7.1` 直接组织 `Pencil` 页面设计稿与截面图。

## 当前结论

1. 本节点主路线已切换为 `Pencil Project + Codex` 直接协同设计。
2. 自动生成工具不再作为当前执行前提。
3. 后续设计工作的关键，不是继续比工具，而是把页面语义、栏目职责、中文命名和信息密度定义清楚。
