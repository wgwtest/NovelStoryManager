# WBS 4.4.1 场景调度工作面要素设计

## 页面构成

1. 左侧常驻对象库
2. 中部场景舞台图
3. 右侧场景检查器
4. 辅助地点调度表与空间切换顺序区

## 核心要求

1. 场景舞台图必须同时表达地点节点、驻留对象和转场方向。
2. 地点卡片要区分主舞台、次级场和临时过渡场。
3. 场景切换顺序必须可见，不允许只靠文字摘要脑补。
4. 场景调度工作面负责空间组织与场景流转，不负责卷宗字段编辑或时间编排。

## 必备能力

1. 地点节点展示
2. 地点节点拖拽
3. 场景切换线展示
4. 驻留对象高亮
5. 场景顺序列表联动
6. 空间冲突提示

## 设计约束

1. 场景调度页要强调空间关系，而不是伪装成另一种时间轴。
2. 驻留对象与地点切换必须能通过颜色、线型或标签区分。
3. 空间冲突和占用重叠要有直接反馈，不能只写在检查器文字里。
