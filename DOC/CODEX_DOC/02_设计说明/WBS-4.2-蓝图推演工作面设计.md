# WBS 4.2 蓝图推演工作面设计

## 工作面定位

`蓝图推演` 是逻辑与关系推演工作面。

之所以使用“蓝图推演”这个名字，而不是泛泛的“关系图”，是因为它不是只负责把对象连成一张图，而是要表达：

1. 节点之间如何触发
2. 关系如何流动
3. 哪些链路成立，哪些链路不成立
4. 一段局部逻辑如何嵌在整体推演中

它的目标更接近可编辑的蓝图式逻辑面，而不是静态关系海报。

## 解决的问题

蓝图推演工作面主要解决三类问题：

1. 对象之间虽然有关联，但因果链与触发链不直观
2. 逻辑路径缺少可编辑的节点化表达
3. 关系存在但难以判断联通性与局部影响范围

因此，它是“推演面”，不是卷宗的图形装饰版。

## 与其他工作面的边界

### 1. 蓝图推演负责什么

1. 节点创建与布局
2. 端口和连线编辑
3. 局部子图聚焦
4. 联通性与合法性提示
5. 节点属性观察与修改入口

### 2. 蓝图推演不负责什么

1. 不负责对象档案总览
2. 不负责多轨时间编排
3. 不负责空间舞台布局
4. 不负责章节切片直接输出

## 数据驱动关系

### 1. 主要输入数据

蓝图推演读取共享对象真源，尤其依赖：

1. `events.json`
2. `relations.json`
3. `characters.json`
4. `clues.json`
5. `arcs.json`

### 2. 视图状态输入

蓝图推演还需要读取和写入专属视图状态，例如：

1. 节点位置
2. 画布视口
3. 当前子图过滤
4. 当前选中节点

首轮设计优先落在 `views/graph-layouts.json` 一类视图状态文件中，不写回对象事实文件。

### 3. 输出动作

蓝图推演输出分为两类：

1. 修改对象之间的正式关联语义
2. 修改节点布局与画布观察状态

前者影响全部工作面，后者只影响蓝图推演本身。

## 页面结构

### 1. 左侧对象库

对象库在这里兼具“对象总览”和“拖入节点来源”的双重作用。

### 2. 中央推演画布

中央画布是蓝图推演的主体，至少包含：

1. 节点层
2. 连线层
3. 端口热点
4. 局部图聚焦区域
5. 联通与错误提示

### 3. 顶部或悬浮工具栏

工具栏负责高频推演动作，例如：

1. 创建节点
2. 缩放
3. 重置视口
4. 切换模式
5. 保存布局

### 4. 右侧检查器

检查器用于编辑当前节点的属性，不承载整图结构。

## 关键交互

1. 从对象库拖入对象，生成节点
2. 从输出端口拖向输入端口，建立连接
3. 悬停、起拖和命中目标时，端口必须给出明显反馈
4. 非法连接必须被明确拒绝
5. 选中节点后，检查器与画布状态同步更新

## 设计约束

1. 节点表达必须贴近“可编排蓝图节点”，不能退化成普通卡片加几条线。
2. 连线与端口反馈必须清晰，不能让用户靠猜测判断能否连接。
3. 蓝图推演的布局、缩放和局部聚焦属于视图状态，不写回对象事实。
4. 蓝图推演与其他工作面共享对象真源，但不共享中央主区实现。
