# WBS 4.1.1 卷宗工作面要素设计

## 设计前提

本文件只回答“卷宗页面由什么构成”，不替代父级文档对“为什么需要卷宗工作面”的解释。父级定位见 `WBS-4.1-卷宗工作面设计.md`。

## 页面构成

1. 左侧常驻对象库
2. 中部卷宗目录与卷宗正文
3. 右侧对象检查器
4. 底部或右下方可展开的审校队列与引用回查区

## 数据映射

1. 对象列表与卷宗正文由 `objects/*.json` 驱动。
2. 搜索、筛选、最近浏览与卷宗展开状态由 `views/*.json` 驱动。
3. 字段编辑应改写对象事实。
4. 卷宗展开、筛选与浏览位置只改卷宗视图状态。

## 核心要求

1. `卷宗目录 / 卷宗摘要 / 审校队列 / 引用回查` 四块都必须稳定可见和可滚动。
2. 引用按钮点击后必须切换对象类型并选中目标对象。
3. 卷宗工作面负责对象浏览、对象创建、字段编辑和卷间核对，不负责蓝图连线或时间编排。

## 要素清单

1. 卷宗标题区：对象名、类型、标签、审校状态
2. 核心字段区：最重要的事实字段与摘要
3. 引用回查区：引用谁、被谁引用
4. 审校队列区：缺失字段、异常提示、待补项
5. 快速动作区：新建对象、跳转引用、切换相邻卷宗
