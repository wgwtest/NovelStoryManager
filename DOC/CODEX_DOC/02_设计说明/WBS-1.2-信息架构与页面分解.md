# WBS 1.2 信息架构与页面分解

## 当前冻结口径

1. 本文保留了早期三视图表述，作为信息架构演进记录。
2. 当前真正生效的中心工作区口径，已经由后续评阅收束为 `卷宗 / 蓝图推演 / 剪辑编排 / 场景调度` 四个工作面。
3. 左侧对象库常驻、右侧检查器常驻这一总结构保持不变。

## 目标

在进入低保真原型和技术选型前，先明确工作台由哪些固定区域构成、三大视图分别承担什么职责、对象编辑从哪里进入。

## 总体工作台结构

首版工作台采用固定三段式结构：

1. 左侧：常驻对象库
2. 中心：工作区与视图标签页
3. 右侧：上下文检查器

辅助区域：

1. 顶部命令栏
2. 视图内工具栏

## 全局壳层分解

### 1. 顶部命令栏

承担全局动作，不承担具体视图编辑。

建议包含：

1. 当前项目名称
2. 打开项目 / 保存项目 / 导入 / 导出
3. 当前视图标签入口
4. 全局搜索入口
5. 快速创建对象入口

### 2. 左侧对象库

这是全局常驻区，不随主视图切换而消失。

建议包含：

1. 对象类型切换
2. 搜索与筛选
3. 对象列表
4. 新建对象
5. 当前对象计数

对象库的职责：

1. 作为对象真源的浏览入口
2. 作为拖拽源，将对象拖入关系图或轨道视图
3. 作为任何视图中的统一选中入口

### 3. 中心工作区

这是主视图承载区，通过标签页切换。

首版标签页固定为：

1. `Knowledge`
2. `Graph`
3. `Tracks`

首版要求：

1. 三个视图可切换显示
2. 三个视图松耦合，各自保存视图状态
3. 所有编辑最终都落回对象库或视图状态文件

### 4. 右侧检查器

右侧检查器是全局上下文区，不是额外主视图。

其职责：

1. 显示当前选中对象详情
2. 显示当前选中关系边详情
3. 显示当前选中轨道块详情
4. 提供字段编辑入口

## 三大视图分解

### Knowledge 视图

职责：

1. 面向结构化对象的查看和编辑
2. 强调字段维护、列表筛选、对象引用跳转

主区域建议：

1. 顶部：当前对象类型工具栏
2. 中部：对象列表或表格
3. 右侧检查器：对象字段详情

### Graph 视图

职责：

1. 面向对象关系的可视化浏览与编辑
2. 强调节点拖拽、边连线、子图观察

主区域建议：

1. 顶部：图视图工具栏
2. 中部：无限画布
3. 左侧对象库：拖拽节点来源
4. 右侧检查器：节点 / 边属性

### Tracks 视图

职责：

1. 面向事件与剧情编排的多轨观察
2. 强调块编排、换轨、分组和观察维度切换

主区域建议：

1. 顶部：轨道分组与缩放工具栏
2. 中部：多轨编排区
3. 左侧对象库：拖拽事件、角色、地点等进入轨道
4. 右侧检查器：轨道块和预设详情

## 编辑入口与保存边界

### 修改对象事实的入口

1. 对象库新建对象
2. Knowledge 视图字段编辑
3. 右侧检查器字段编辑
4. Graph / Tracks 中显式触发的“编辑对象”动作

### 只修改视图状态的入口

1. Graph 节点布局调整
2. Tracks 轨道显示分组
3. 视图缩放级别
4. 当前筛选条件

## 首版不做的界面复杂度

1. 不做多窗口或多工作区布局
2. 不做多实例同类视图标签
3. 不做正文编辑工作区
4. 不做数据库或协同相关控制面板
5. 【已确认，没什么问题】
