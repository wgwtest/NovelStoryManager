## 三栏语义说明

### 左列：对象库

1. 左列是真实对象的导航入口，不是装饰性边栏。
2. 它承担四件事：
   - 搜索当前对象
   - 切换对象分类
   - 查看对象列表
   - 触发快速创建
3. 推荐信息顺序：
   - 搜索框
   - 分类切换
   - 计数与快捷动作
   - 对象列表

### 中列：主视图工作区

1. 中列是当前 tab 的主要工作面。
2. `知识库` 的含义：
   - 对结构化对象真值进行浏览、筛选、排序和编辑
   - 它不是章节写作界面
   - 它不是关系图和轨道图
3. `关系图` 的含义：
   - 对对象关系、依赖和连接结构进行可视化编辑
4. `轨道` 的含义：
   - 对时间、空间和观察切片进行多轨编排

### 右列：检查器

1. 右列不是任意堆字段的杂项区。
2. 它存在的理由是让用户在不离开当前主视图的情况下，看到“当前选择对象的辅助详情”。
3. 它应承载：
   - 当前对象身份信息
   - 基础字段与扩展字段
   - 引用关系与跳转
   - 当前视图相关的辅助说明
4. 它不应承担与中列重复的大面积主内容展示。

## 对象库分类约束

对象库至少覆盖以下对象类型，并提供中文标签：

1. `characters` -> 人物
2. `factions` -> 势力
3. `locations` -> 地点
4. `items` -> 物品
5. `realm-systems` -> 境界体系
6. `events` -> 事件
7. `relations` -> 关系
8. `clues` -> 线索
9. `arcs` -> 剧情弧

## 布局与密度规则

1. 顶栏产品标题小型化，不再展示夸张的大标题。
2. 样例项目名只做次级信息提示。
3. tab 区域改为紧凑横排，不再让单个 tab 占用过大高度。
4. 左列卡片、按钮、标签和筛选控件整体缩紧，提升工作台密度。
5. 页内术语与按钮统一中文化。
6. 整体背景改为白色中性工作台，而不是暖黄底。
