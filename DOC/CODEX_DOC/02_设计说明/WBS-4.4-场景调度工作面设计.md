# WBS 4.4 场景调度工作面设计

## 工作面定位

`场景调度` 是空间组织工作面。

之所以单独建立这一工作面，而不是把地点信息塞进轨道或关系图，是因为它回答的问题本质上是空间问题：

1. 谁在哪个地点
2. 场景如何切换
3. 空间路径怎么走
4. 哪些地点发生冲突或重叠

它不是时间轴的另一种皮肤，而是舞台与场域的组织面。

## 解决的问题

场景调度主要解决以下问题：

1. 地点关系难以直观看出
2. 角色和事件的空间驻留不清楚
3. 转场路径与空间冲突缺少可视表达
4. 时间编排完成后，仍缺少空间层面的单独观察面

## 与其他工作面的边界

### 1. 场景调度负责什么

1. 地点组织
2. 驻留对象观察
3. 转场路径表达
4. 空间冲突提示
5. 空间顺序与调度说明

### 2. 场景调度不负责什么

1. 不负责对象档案编辑
2. 不负责逻辑因果推演
3. 不负责时间线编排
4. 不负责章节文本本身

## 数据驱动关系

### 1. 主要输入数据

场景调度依赖共享对象真源，重点包括：

1. `locations.json`
2. `characters.json`
3. `events.json`
4. `relations.json`

### 2. 视图状态输入

场景调度还需要专属视图状态，例如：

1. 场景舞台布局
2. 当前空间过滤条件
3. 当前驻留高亮对象
4. 当前调度顺序

这些状态应存入 `views/*.json`，不写回地点对象事实。

### 3. 输出动作

场景调度输出的是空间观察结果和布局状态；只有在用户显式修改地点关系或事件归属时，才会写回对象真源。

## 页面结构

### 1. 左侧对象库

左侧对象库负责把地点、人物、事件等对象送入场景调度上下文。

### 2. 中央视台区

场景调度的中央主区至少包含：

1. 地点节点
2. 转场连线
3. 驻留对象标记
4. 场景层级或分区

### 3. 辅助调度区

辅助区用于承载：

1. 地点调度表
2. 空间切换顺序
3. 冲突与占用提示

### 4. 右侧检查器

检查器用于查看当前地点或转场的属性与限制。

## 关键交互

1. 选中地点时，高亮相关驻留对象和转场路径
2. 拖动地点节点时，舞台布局更新
3. 查看某条转场时，检查器与调度表联动
4. 空间冲突出现时，中央区和辅助区都要给出明确提示

## 设计约束

1. 场景调度必须首先表达空间组织，不能回退成另一种时间表。
2. 地点节点、驻留对象和转场必须在视觉上可区分。
3. 布局变更属于视图状态，地点事实属于对象真源，两者必须分开写入。
4. 场景调度与其他工作面共享数据真源，但可以独立替换其中心展示实现。
