# WBS 1.7 中高保真原型设计与生成方案

## 目标

在现有低保真原型与交互验证的基础上，补出一套“自动生成草稿 + 人工精修设计稿”并行的中高保真原型方案，用于快速形成三主视图的视觉与交互基线。

## 新增约束

1. `Knowledge`、`Graph`、`Tracks` 三视图都需要能快速生成更接近最终形态的界面草稿。
2. `Graph` 视图节点不能再停留在普通矩形卡片，应朝 Unreal 蓝图式节点靠拢：
   - 左侧输入端口
   - 右侧输出端口
   - 分类型参数区
   - 端口可吸附、可高亮
   - 连线存在明确的有效/无效反馈
3. 自动生成产物是原型基线，不直接作为最终代码或最终设计稿。
4. 对工作台局部高信息密度区域，需要允许用人工工具单独补出截面图与页面设计稿。

## 候选方案对比

### 方案 A：`Uizard Autodesigner`

优点：

1. 官方定位就是“用文本快速生成多页面、可编辑的 UI/原型草稿”。
2. 更适合作为“先出一版中高保真界面，再人工校正”的起点。
3. 对三主视图这种多页面工作台结构更直接。

不足：

1. 对强交互编辑器内部的节点细节，需要二次人工精修。
2. 生成出的图形编辑器局部，通常只能先给出视觉方向，不能直接等同最终交互定义。

### 方案 B：`Visily`

优点：

1. 官方强调可把文本、截图或草图转成可编辑的 UI 设计。
2. 如果后续要把当前已有页面截图转成更精细的界面草稿，它比纯文本生成更方便。
3. 适合作为“先有页面骨架，再做界面提纯”的补充方案。

不足：

1. 更依赖已有参考图、线框或截图。
2. 对复杂节点式编辑器，同样需要后续人工补端口、锚点和状态细节。

### 方案 C：`Figma AI Prototype / Make`

优点：

1. 官方强调可在同一工具内快速生成高保真原型与交互流。
2. 适合作为交互动线和视觉气质的快速探索工具。

不足：

1. 它更适合做交互探索，不应当单独承担最终“可维护原型资产”的唯一来源。
2. 对本项目这种多视图、编辑器式界面，仍需要结构化输入和人工修正。

### 方案 D：`Pencil Project`

优点：

1. 本地化、文件化、开源，适合作为稳定的手工设计稿与截面图工具。
2. 适合承接对象库截面、关系图局部节点区、多轨局部时间区等高信息密度页面草稿。
3. 当自动生成工具已经产出整页草稿后，`Pencil` 更适合作为人工精修补充路线。

不足：

1. 它不承担“自动生成多页面高保真草稿”的核心能力，不能直接替代当前主方案。
2. 更偏手工排版与拼装，不适合作为首轮多方案生成器。
3. 对端口吸附、动态连线、交互流等高交互目标，只适合表达静态设计稿，不适合作为唯一原型来源。

## 推荐路线

### 主推荐

`Uizard Autodesigner` 作为首选生成器。

原因：

1. 它更贴合“先生成多页面中高保真界面草稿，再人工做正式原型”的目标。
2. 它对 `Knowledge / Graph / Tracks` 这类多页面工作台更合适。
3. 它不要求先有现成截图，适合从当前文档约束直接生成第一版草稿。

### 备选补强

`Visily` 作为补强方案。

使用时机：

1. 需要把当前运行中的界面截图转成可编辑设计时。
2. 需要在已有结构基础上继续提高视觉质量时。

### 交互探索补充

`Figma AI Prototype / Make` 用于交互动线与高保真演示探索，不作为唯一设计源。

### 手工设计稿补充

`Pencil Project` 用于把自动生成草稿收束成可审阅的截面图与页面设计稿，不取代主生成路线。

## 推荐执行流程

1. 先整理生成输入包：
   - 产品边界摘要
   - 三视图结构说明
   - 关键交互约束
   - `Graph` 的蓝图节点风格约束
2. 用 `Uizard Autodesigner` 生成三主视图的第一版中高保真草稿。
3. 对 `Graph` 视图单独补充节点样式要求，再生成一版偏 Unreal 蓝图风格的关系图工作台。
4. 若需要把当前实现界面作为参考，再把截图交给 `Visily` 做二次提纯。
5. 对对象库、关系图节点区、多轨区等局部高信息密度区域，使用 `Pencil Project` 产出截面图与页面设计稿。
6. 最终把筛选出的方案人工整理为正式高保真原型。

## 生成输入包模板

### 通用输入

1. 左侧常驻对象库
2. 中间主工作区按 `Knowledge / Graph / Tracks` 切换
3. 右侧检查器或属性面板
4. 顶部保留项目级标题、保存状态和视图控制
5. 允许拆出“整页草稿”和“局部截面图”两类输出

### `Knowledge` 视图输入补充

1. 对象列表区
2. 对象详情区
3. `Core Fields / References / Additional Fields` 三段式分组
4. 引用跳转与分组折叠

### `Graph` 视图输入补充

1. 节点风格参考 Unreal 蓝图节点
2. 节点包含标题条、类型标签、左输入端口、右输出端口、参数区、状态标记
3. 连线要有吸附感、选中高亮和非法连接反馈
4. 工具栏包含创建节点、缩放、画布尺寸和坐标显示
5. 背景允许有网格，但节点和端口必须清晰可点

### `Tracks` 视图输入补充

1. 类似剪辑软件多轨界面
2. 左侧轨道头，右侧轨道内容区
3. 时间尺、空间线、分组切换、预设管理
4. 章节观察位与时间覆盖关系提示

## 输出要求

1. 至少生成 `Knowledge / Graph / Tracks` 三张主界面草稿。
2. `Graph` 草稿至少要给出一版“蓝图节点强化版”。
3. 草稿必须保留对象库、主工作区、右侧检查器三段式结构。
4. 草稿要能支撑后续高保真原型制作，而不是仅有一张营销图。
5. 局部高保真截面图允许由 `Pencil Project` 单独补绘，不强制全部由同一生成器一次完成。

## WBS 拆分补充

1. `WBS 1.7` 保持为父节点，负责冻结：
   - 主生成路线
   - 补充路线边界
   - 输入包模板
   - 输出要求
2. 其下新增两个子节点：
   - `WBS 1.7.1 中高保真原型草稿方案细化`
   - `WBS 1.7.2 Pencil截面图与页面设计稿`
3. 分工原则：
   - 自动生成工具负责快速形成整页草稿
   - `Pencil Project` 负责补齐局部截面图和人工精修设计稿
4. `Pencil` 的定位是补充路线，而不是对当前主推荐路线的回退替换。

## 选型结论

1. 主方案：`Uizard Autodesigner`
2. 补充方案：`Visily`
3. 交互演示补充：`Figma AI Prototype / Make`
4. 手工截面图与页面设计稿补充：`Pencil Project`
